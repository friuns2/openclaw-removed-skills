# 评估规则与日历推算

## 日历推算规则

skill 在生成报告时必须进行日历推算：

1. 获取当前日期
2. 确定用户提到的节假日的具体日期范围
3. 确定节假日前后的工作日和周末分布
4. 计算拼假方案（请 X 天假可获得连续 Y 天假期）
5. 标注具体的星期几
6. 所有日期建议必须精确到"X 月 X 日（周X）"

## 拥挤风险评估逻辑

评估节假日拥挤程度时，参考以下维度：

### 1. 节假日等级

| 等级 | 拥挤程度 | 对应节假日 |
|------|----------|------------|
| S 级 | 极高拥挤 | 国庆、春节 |
| A 级 | 高拥挤 | 五一 |
| B 级 | 中等拥挤 | 端午、中秋 |
| C 级 | 较低拥挤 | 清明、元旦 |

### 2. 目的地热度分级

**一线热门（拥挤高、溢价高）**：三亚、丽江、厦门、成都、西安、杭州、重庆、北京、大理、桂林

**二线特色（体验好、性价比高）**：泉州、潮汕、长沙、贵阳、洛阳、南京、福州、昆明、青岛、武汉、扬州、太原

**三线小众（人少、独特体验）**：平潭、荔波、腾冲、恩施、阆中、甘南、霞浦、柳州、泸沽湖、镇远、台州、景德镇

### 3. 出行日期对拥挤指数的影响

| 出行策略 | 拥挤指数变化 | 成本变化 |
|----------|-------------|----------|
| 假期首日出发 | +40% | +20-30% |
| 假期末日返程 | +50% | +15-25% |
| 提前 1 天出发（需请假） | -30% | -25% |
| 推迟 1 天返程（需请假） | -25% | -20% |
| 选择二线替代目的地 | -50~70% | -30~50% |
| 选择三线小众目的地 | -70~90% | -40~60% |

## 输出风格规则

整体风格：
- 像一份决策报告，不像攻略帖
- 明确、果断、有判断，不模棱两可
- 少废话，高信息密度
- 强可执行性：用户看完就知道该怎么做
- 不种草、不煽情、不鸡汤
- 带有"裁决"语气：替用户拿主意，而不是给一堆选项

语气参考：
- 不要说"你可以考虑去 XX"，要说"建议去 XX，不建议去 YY"
- 不要说"XX 也是不错的选择"，要说"XX 是这次最优解，YY 是备选"
- 不要说"看你个人偏好"，要说"基于你的预算/时间，推荐 XX"
