# 主动式出行陪伴技能 v3.0（父母视角）

## 功能概述

**定位**：AI 作为父母的贴身出行助手，主动关怀、实时监控、紧急救助

**核心能力**：
- ✅ **健康监护与紧急救助**（最高优先级）⭐
- ✅ 定时任务调度（起飞前 24 小时确认等）
- ✅ 智能外呼机器人（联系服务商，无需父母操心）
- ✅ 航班动态监控
- ✅ 行程变更处理与方案推荐
- ✅ 每日行程提醒推送（早晚安 + 天气 + 穿衣建议）
- ✅ 旅拍预约协调
- ✅ **迷路定位与接应**⭐
- ✅ **防诈骗预警干预**⭐

**设计原则**：
- 主动不被动：AI 主动询问，而非等待父母求助
- 预防优于处置：提前识别风险，提前介入
- 分级响应：轻微/中等/严重三级处置流程
- 多方联动：地接/子女/医院/保险协同

## 核心功能模块

### 模块 1: 健康监护与紧急救助 ⭐

#### 健康监测指标

```yaml
health_monitoring:
  vital_signs:
    - heart_rate (心率) - 通过智能手表采集
    - blood_pressure (血压) - 手动输入或设备同步
    - blood_sugar (血糖) - 糖尿病患者必填
    - body_temperature (体温) - 可选
    
  symptom_detection:
    - dizziness (头晕)
    - chest_pain (胸痛)
    - shortness_of_breath (呼吸困难)
    - nausea (恶心)
    - fatigue (乏力)
    
  chronic_conditions:
    - hypertension (高血压)
    - diabetes (糖尿病)
    - heart_disease (心脏病)
    - asthma (哮喘)
```

#### 三级响应体系

**Level 1: 轻微不适**
```
触发条件:
- 父母自述"有点累"、"不太舒服"
- 心率/血压轻微异常（超出正常值 10-20%）
- 轻度头晕、乏力

处置流程:
1. AI 安抚情绪，建议休息
2. 通知地接导游前来陪同
3. 调整后续行程（减缓节奏）
4. 持续观察 30 分钟

回复模板:
"阿姨您先别急，请您先在休息区坐下。
我马上联系导游小李过来陪您。
今天的行程我已经调整为慢节奏，不着急。
您现在感觉怎么样？有没有好一点？"
```

**Level 2: 中等症状**
```
触发条件:
- 父母自述"很难受"、"疼痛"
- 心率/血压显著异常（超出正常值 20-50%）
- 持续头晕、呕吐、胸闷

处置流程:
1. AI 立即通知地接社安排车辆送医
2. 同步通知子女
3. 调取最近医院信息
4. 启动保险理赔预登记
5. 调整后续行程

回复模板:
"叔叔您先别动，我马上安排人过来。
已经通知旅行社派车送您去最近的医院。
同时也通知了您的儿子/女儿。
医院地址是：{hospital_name}，距离 2 公里。
医生会在门口等您，不要担心。"
```

**Level 3: 严重紧急情况**
```
触发条件:
- 父母自述"救命"、"不行了"
- 意识模糊、无法回应
- 心率/血压危急值（超出正常值 50% 以上）
- 跌倒、外伤出血

处置流程:
1. AI 自动拨打 120 急救电话
2. 同时通知地接社派人到现场
3. 通知子女立即赶往医院
4. 调取健康档案（病史、过敏史）
5. 启动保险紧急救援
6. 联系大使馆（境外旅行）

黄金 5 分钟流程:
- 0-30 秒：AI 冷静安抚，判断紧急程度
- 30 秒 -2 分钟：分级处置（轻微/中等/严重）
- 2-5 分钟：联动地接/子女/医院/保险
- 后续：调整行程 + 持续跟进

回复模板:
"叔叔/阿姨，我已经拨打 120 了，救护车马上就到。
请您尽量保持平静，不要乱动。
导游小王正在赶过来，5 分钟内到。
您的孩子也已经通知了。
救护车会去最近的 {hospital_name} 医院。
我会一直陪着您，直到您安全为止。"
```

### 模块 3: 行程变更处理

#### 变更场景分类

| 变更原因 | 影响范围 | 处理流程 |
|---------|---------|---------|
| **天气原因** | 户外活动取消 | 启动室内备选方案 |
| **交通延误** | 后续行程推迟 | 联系服务商调整时间 |
| **健康问题** | 部分/全部取消 | 医疗优先，调整行程 |
| **个人原因** | 自愿放弃 | 协助退改签 |

#### 变更处理流程

```python
def handle_itinerary_change(reason, affected_activities):
    # Step 1: 评估影响
    impact = assess_impact(reason, affected_activities)
    
    # Step 2: 生成备选方案
    alternatives = generate_alternatives(affected_activities)
    
    # Step 3: 联系服务商确认可用性
    for alt in alternatives:
        availability = contact_providers(alt)
        if availability:
            break
    
    # Step 4: 向父母呈现方案
    present_options(alternatives)
    
    # Step 5: 确认后执行变更
    if user_confirms():
        execute_change(selected_option)
        notify_child()  # 通知子女
        update_insurance()  # 更新保险信息
    
    return selected_option
```

好的阿姨，根据您的描述，可能是低血糖或者疲劳引起的。

您现在感觉怎么样？有没有好一点？
