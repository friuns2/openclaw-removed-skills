# elm Skill

`elm` 已经从“饿了么红包真假价值判断”升级成“饿了么下单协助”。

它现在可以接住用户的自然语言需求，要求用户提供送货地址、商品名、具体要求和预算；如果用户还没登录，skill 会把用户带到饿了么官方登录页面并等待用户自行完成登录，然后继续按地址搜索附近可送达商品，比较优惠券、红包、满减、配送费和包装费，最后把最优购物车准备好，提醒用户去支付。

## 适合什么问题

- “帮我点一份送到这个地址的黄焖鸡，少辣，不要香菜，预算 35 元以内。”
- “我先自己登录饿了么，你再帮我看看附近有没有无糖奶茶优惠后最划算的。”
- “按这个地址搜水果和矿泉水，尽量用掉红包，帮我加到购物车。”
- “想吃夜宵，预算 50 元以内，优先快送达，直接帮我把购物车配好。”

## 它会怎么帮用户

默认会完成这条路径：
- 收集地址、商品、要求、预算
- 如未登录，等待用户在官方页面自行完成登录
- 根据地址搜索附近商家和商品
- 比较真实到手价而不是只看表面优惠
- 选择最优商家和购物车方案
- 加入购物车
- 通知用户去支付

## 它重点优化什么

- 可见红包、优惠券、满减、配送费、包装费后的真实应付价
- 送达地址对应的可配送范围和 ETA
- 商品要求是否真的被满足，比如少辣、不要香菜、品牌偏好、无糖、规格限制
- 预算内的最优购物车，而不是只找最低标价
- 需要凑单时是否值得，为了省几块钱加无用商品会被直接识别为坏方案

## 典型输出

- `Cart Ready Status`
- `Selected Merchant And Basket`
- `Savings Summary`
- `Payment Handoff`

## 安全边界

允许在用户明确同意下执行：
- 使用用户已完成认证的饿了么会话
- 打开官方登录页并等待用户自行完成登录
- 读取可见地址、购物车、红包和优惠信息
- 搜索商品并加入购物车

不会执行：
- 自动读取站外消息或通知
- 要求用户把登录秘密信息发进聊天
- 代用户输入认证信息
- 保存地址或账号数据
- 代用户完成支付
- 提交不可逆订单

## 安装

```bash
clawhub install elm
```
