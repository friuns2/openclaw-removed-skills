# 学术论文写作工作流（Academic Paper Workflow）

## 🎯 核心价值

**这不是单个技能，而是一套完整的 AI 科研协作流程。**

基于用户（纪老师）29 天实战验证，产出 1 篇 SCI 论文（182 篇文献检索 → 96 篇引用 → 2 轮诚信检查 → 3 轮同行评审 → 0 次手动调格式）。

### 核心特点

| 特点 | 说明 |
|------|------|
| **透明** | 每个技能职责明确、每阶段产出清晰、每决策有记录 |
| **可审核** | 研究计划需审批、诚信检查 2 轮、评审 5 视角、格式有模板 |
| **可追溯** | 每引用有来源（7 数据库）、每修改有版本、每结论有证据 |
| **无幻觉** | 强制 2 轮研究、2 轮诚信检查、3 轮同行评审 |

---

## 📋 工作流总览

### 7 阶段 + 8 技能

```
阶段 1: Research（深度研究）
  └─ academic-deep-research → 25000 字研究报告 + 文献搜索结果

阶段 2: Write（学术写作）
  └─ academic-writer → 15000 字论文初稿

阶段 3: Citation（引用格式化）
  └─ academic-citation → 96 篇 APA 7th 参考文献

阶段 4: Integrity Check（初稿诚信检查）
  └─ academic-integrity-checker → 发现并修复错误引用

阶段 5: Review → Revise → Re-Review（多轮评审）
  └─ academic-review-revisor → 18 条意见 → 4 条 → 0 条

阶段 6: Final Integrity Check（终稿验证）
  └─ final-integrity-check → 投稿前最终验证

阶段 7: Journal Format Output（期刊格式输出）
  └─ journal-format-output → 完全符合期刊格式的投稿文件
```

### 技能分工

| 角色 | 技能名称 | 来源 | 核心职责 |
|------|---------|------|---------|
| 文献检索专家 | academic-search | 🔧 自研 | 7 个国内外文献数据库集成检索 |
| 深度研究员 | academic-deep-research | 官方 | 两轮研究，强制严谨 |
| 学术写手 | academic-writer | 官方 | 基于搜索结果，按模板生成论文 |
| 引用格式化师 | academic-citation | 官方 | 支持多种引用格式 |
| 同行评审员 | academic-review-revisor | 🔧 自研 | 多轮评审 + 自动润色 |
| 诚信检查员 | academic-integrity-checker | 官方 | 初稿验证 |
| 最终审核员 | final-integrity-check | 🔧 自研 | 投稿前最终验证 |
| 格式输出专家 | journal-format-output | 🔧 自研 | 按期刊模板自动格式化 |

## 🚀 快速启动

### 方式 1：完整工作流（推荐首次使用）

```bash
/run-academic-workflow [论文主题] --journal [目标期刊] --depth exhaustive
```

这将自动执行全部 7 个阶段，每阶段完成后等待你确认。

### 方式 2：分阶段执行（推荐有经验后）

```bash
# 阶段 1: 深度研究
/academic-deep-research [你的主题]

# 阶段 2: 学术写作
/academic-writer 基于以下研究报告撰写论文 [粘贴研究报告]

# 阶段 3: 引用格式化
/academic-citation 将参考文献转换为 APA 7th 格式

# 阶段 4: 初稿诚信检查
/academic-integrity-checker [论文初稿]

# 阶段 5: 同行评审 + 润色（多轮）
/review-revisor [论文稿]

# 阶段 6: 终稿验证
/final-integrity-check [最终稿]

# 阶段 7: 期刊格式输出
/journal-format-output [论文稿] --journal "目标期刊名称"
```

## 📖 详细阶段说明

### 阶段 1: Research（深度研究）

**目标：** 生成 25000 字深度研究报告，为写作奠定基础。

**核心技能：** `academic-deep-research`

**执行流程：**

1. **启动研究**
   ```bash
   /academic-deep-research [你的研究主题]
   ```

2. **回答澄清问题**（技能会主动询问）
   - 应用场景（军事/民用/混合）
   - 研究深度（概述/exhaustive）
   - 时间/地域/来源偏好

3. **审批研究计划**（技能生成 6 个主题的研究计划）
   - 检查主题覆盖是否完整
   - 确认关键问题是否准确
   - 批准或调整

4. **执行 2 轮研究**（每主题）
   - **第 1 轮：全景扫描**
     - 搜索 arXiv 等数据库（≥20 条结果）
     - 识别关键来源、主要玩家、核心概念
     - 记录知识缺口和矛盾点
   
   - **第 2 轮：深度挖掘**
     - 针对缺口定向搜索
     - 深入提取原始来源
     - 挑战第 1 轮假设，寻找反例

5. **输出**
   - 25000 字深度研究报告
   - 文献搜索结果（182 篇左右）
   - 思考演进记录

**⏱ 预计耗时：** 2-4 小时

**✅ 完成标准：**
- [ ] 6 个主题全部完成 2 轮研究
- [ ] 每主题有明确结论和证据
- [ ] 知识缺口和矛盾点已标注
- [ ] 文献搜索结果已保存

**📌 关键决策点：**
- 研究计划审批（必须人工确认）
- 研究深度选择（概述 vs exhaustive）

---

### 阶段 2: Write（学术写作）

**目标：** 将研究报告转化为结构化学术论文初稿。

**核心技能：** `academic-writer`

**执行流程：**

1. **启动写作**
   ```bash
   /academic-writer 基于以下研究报告，撰写一篇 SCI 论文格式的文献综述
   [粘贴阶段 1 的研究报告]
   ```

2. **确认论文结构**
   - 摘要、引言、方法、结果、讨论、结论
   - 图表数量和位置
   - 参考文献数量目标

3. **生成初稿**
   - 提取核心论点和证据
   - 组织成学术论文结构
   - 生成符合学术规范的表述

4. **输出**
   - 15000 字论文初稿
   - 图表草稿
   - 参考文献列表（未格式化）

**⏱ 预计耗时：** 30-60 分钟

**✅ 完成标准：**
- [ ] 论文结构完整（摘要→结论）
- [ ] 核心论点清晰
- [ ] 每论点有证据支持
- [ ] 参考文献列表完整

**📌 关键决策点：**
- 论文结构确认
- 目标期刊选择（影响后续格式）

---

### 阶段 3: Citation（引用格式化）

**目标：** 将参考文献批量转换为目标期刊要求的格式。

**核心技能：** `academic-citation`

**执行流程：**

1. **启动格式化**
   ```bash
   /academic-citation 将参考文献转换为 APA 7th 格式
   [粘贴参考文献列表]
   ```

2. **选择格式**
   - APA 7th（默认）
   - IEEE
   - Chicago
   - 目标期刊特定格式

3. **批量处理**
   - DOI 导入，自动补全元数据
   - 格式验证（作者姓名、期刊名缩写、卷期页码、DOI 链接）
   - 一次性格式化 90+ 篇参考文献

4. **输出**
   - 格式化后的参考文献列表
   - 格式验证报告

**⏱ 预计耗时：** 10-20 分钟

**✅ 完成标准：**
- [ ] 所有参考文献格式统一
- [ ] DOI 链接完整有效
- [ ] 格式验证通过

**📌 关键决策点：**
- 引用格式选择（根据目标期刊）

---

### 阶段 4: Integrity Check（初稿诚信检查）

**目标：** 检测并修复 AI 幻觉、虚假引用、伪造内容。

**核心技能：** `academic-integrity-checker`

**执行流程：**

1. **启动检查**
   ```bash
   /academic-integrity-checker [论文初稿]
   ```

2. **逐条验证**
   - AI 幻觉检测
   - 虚假引用检测
   - 伪造内容检测
   - 引用不匹配检测

3. **检查报告**
   ```
   ✅ 验证通过：93 条引用
   ⚠️ 需要核实：2 条引用（来源模糊）
   ❌ 发现错误：1 条引用（文献不存在）
   ```

4. **人工修复**
   - 删除错误引用
   - 补充真实文献
   - 核实模糊来源

**⏱ 预计耗时：** 30-60 分钟（含人工修复）

**✅ 完成标准：**
- [ ] 所有引用验证通过
- [ ] 无 AI 幻觉
- [ ] 无虚假引用

**📌 关键决策点：**
- 模糊引用的处理（删除/替换/保留）

---

### 阶段 5: Review → Revise → Re-Review（多轮评审）

**目标：** 通过 5 视角评审 + 自动润色，达到 0 评审意见。

**核心技能：** `academic-review-revisor`

**执行流程：**

1. **第 1 轮评审**
   ```bash
   /review-revisor [论文稿]
   ```

2. **5 视角评审意见**
   - **方法论专家：** 搜索策略、PRISMA 流程图、方法描述
   - **领域专家：** 技术进展、对比分析、领域覆盖
   - **统计专家：** 指标说明、测试集规模、置信区间
   - **写作专家：** 摘要长度、语言流畅度、逻辑结构
   - **伦理专家：** 数据集来源、隐私合规、利益冲突

3. **自动润色**
   ```bash
   /academic-writer 基于以下评审意见修改论文：[粘贴评审意见]
   ```

4. **第 2 轮评审**
   - 检查第 1 轮问题是否修复
   - 发现遗留小问题（通常 3-5 条）

5. **第 2 轮润色**
   - 修复遗留问题

6. **第 3 轮评审（最终确认）**
   - 目标：0 评审意见

**⏱ 预计耗时：** 2-4 小时（含多轮迭代）

**✅ 完成标准：**
- [ ] 5 视角评审全部通过
- [ ] 评审意见 = 0 条
- [ ] 论文质量达到投稿标准

**📌 关键决策点：**
- 评审意见的优先级排序
- 是否接受某些评审建议（需人工判断）

---

### 阶段 6: Final Integrity Check（终稿验证）

**目标：** 投稿前最终验证，确保万无一失。

**核心技能：** `final-integrity-check`

**执行流程：**

1. **启动终稿检查**
   ```bash
   /final-integrity-check [最终稿]
   ```

2. **验证项目**
   - ✅ 所有引用是否均可追溯
   - ✅ 数据是否与来源一致
   - ✅ 图表是否都有正确标注
   - ✅ 利益冲突声明是否完整
   - ✅ 数据集来源声明是否规范
   - ✅ 是否符合目标期刊基本要求

3. **输出报告**
   ```
   ✅ 验证通过：96 条引用
   ✅ 所有引用均可追溯
   ✅ 未发现 AI 幻觉
   ✅ 数据与来源一致
   ✅ 图表标注完整
   ✅ 声明文件齐全
   ```

**⏱ 预计耗时：** 20-30 分钟

**✅ 完成标准：**
- [ ] 所有验证项目通过
- [ ] 无遗留问题
- [ ]  ready for submission

**📌 关键决策点：**
- 无（纯验证，无需决策）

---

### 阶段 7: Journal Format Output（期刊格式输出）

**目标：** 自动生成完全符合目标期刊格式的投稿文件。

**核心技能：** `journal-format-output`

**执行流程：**

1. **启动格式输出**
   ```bash
   /journal-format-output [论文稿] --journal "目标期刊名称"
   ```

2. **自动处理**
   - 加载期刊模板（Word/LaTeX）
   - 转换文档格式（字体、字号、行距、页边距）
   - 调整参考文献（按期刊要求重新格式化）
   - 重排图表位置（符合期刊规范）
   - 生成投稿文件（主文档、补充材料、封面信）

3. **输出**
   - 主文档（完全符合期刊格式）
   - 补充材料（如有）
   - 封面信（Cover Letter）
   - 投稿检查清单

**⏱ 预计耗时：** 10-20 分钟

**✅ 完成标准：**
- [ ] 格式完全符合期刊要求
- [ ] 所有投稿文件齐全
- [ ] 0 次手动调格式

**📌 关键决策点：**
- 目标期刊确认

## 🔧 自定义与扩展

### 添加新技能

如果工作流缺少某个环节，可以自研技能：

1. **分析需求**：明确技能职责和输入输出
2. **创建技能**：参考 `skill-creator` 技能
3. **集成到工作流**：在适当阶段调用

### 调整阶段顺序

根据论文类型调整：
- **综述论文：** 阶段 1-2 是核心
- **实验论文：** 增加"实验设计"阶段
- **方法论论文：** 增加"方法验证"阶段

### 期刊模板库

在 `assets/journal-templates/` 目录添加目标期刊模板：
```
assets/journal-templates/
├── ieee-transactions.dotx
├── elsevier-jse.dotx
├── springer-mssp.dotx
└── ...
```

## ⚠️ 重要注意事项

### 必须人工参与的环节

| 环节 | 原因 |
|------|------|
| 研究计划审批 | AI 可能遗漏关键主题 |
| 关键决策点确认 | 需要领域专家判断 |
| 模糊引用处理 | 需要人工核实 |
| 评审意见优先级 | 需要判断哪些建议采纳 |
| 最终投稿确认 | 对外发布必须人工确认 |

### 常见陷阱

| 陷阱 | 症状 | 解决方案 |
|------|------|---------|
| 跳过诚信检查 | 投稿后被拒 | 必须执行 2 轮检查 |
| 跳过同行评审 | 论文质量不稳定 | 必须执行 3 轮评审 |
| 手动调格式 | 耗时且易错 | 使用 journal-format-output |
| 研究深度不足 | 论文缺乏深度 | 选择 exhaustive 模式 |

### 时间规划

| 阶段 | 预计耗时 | 可并行 |
|------|---------|-------|
| 阶段 1: Research | 2-4 小时 | ❌ |
| 阶段 2: Write | 30-60 分钟 | ❌ |
| 阶段 3: Citation | 10-20 分钟 | ✅ 可与阶段 4 并行 |
| 阶段 4: Integrity Check | 30-60 分钟 | ✅ 可与阶段 3 并行 |
| 阶段 5: Review/Revise | 2-4 小时 | ❌ |
| 阶段 6: Final Check | 20-30 分钟 | ✅ 可与阶段 7 并行 |
| 阶段 7: Format Output | 10-20 分钟 | ✅ 可与阶段 6 并行 |

**总计：** 6-12 小时（可分多天完成）

## 💡 最佳实践

1. **不要跳过任何阶段**：每个阶段都有独特价值
2. **诚信检查必须 2 轮**：初稿 + 终稿，缺一不可
3. **同行评审至少 3 轮**：直到 0 评审意见
4. **研究计划必须审批**：这是研究方向正确性的保证
5. **最终投稿必须人工确认**：对外发布零容忍错误
