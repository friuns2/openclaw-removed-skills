# 质量控制标准

## 数据质量标准

### 完整性检查

- [ ] 四个核心章节均已覆盖（宏观、资产配置、行业、个股）
- [ ] 每个章节至少包含 1 个表格或要点列表
- [ ] 所有表格字段已填充，无空值
- [ ] 时间范围标注清晰

### 准确性检查

- [ ] 所有观点可溯源至具体券商研报
- [ ] 股票代码、名称匹配正确
- [ ] 评级用语规范（买入/增持/中性/减持/卖出）
- [ ] 数据单位标注清晰（亿元、%，等）

### 时效性检查

- [ ] 所有研报发布日期在时间参数范围内
- [ ] 优先采用最新发布的研报
- [ ] 滞后数据已标注时效风险
- [ ] 无过期数据作为核心依据

---

## 输出质量标准

### 语言风格

- [ ] 专业、客观、简洁
- [ ] 无个人情感表达
- [ ] 无投资建议表述
- [ ] 无 MCP 工具/接口名称暴露

### 格式规范

- [ ] Markdown 格式正确
- [ ] 表格对齐整齐
- [ ] 标题层级清晰
- [ ] 日期格式统一（YYYY 年 MM 月 DD 日）

### 信息密度

- [ ] 每段文字 2-3 句话，无冗余
- [ ] 核心观点突出，无废话
- [ ] 表格内容精炼，每格不超过 50 字
- [ ] 整体篇幅适中（2000-4000 字）

---

## 错误处理标准

### 数据不足场景

| 场景 | 处理方式 | 标注语 |
|------|----------|--------|
| 某章节数据<2 条 | 改用要点列表 | 📋 数据样本不足，暂以摘要形式呈现 |
| 某行业无覆盖 | 标注无数据 | 暂无相关研报覆盖 |
| 全部数据为空 | 告知用户并建议调整时间范围 | 指定时间范围内无相关研报，建议放宽至近 5 日 |

### 技术故障场景

| 场景 | 处理方式 | 告知用户 |
|------|----------|----------|
| mcporter 未安装 | 引导安装 | 请先安装 mcporter：npm install -g mcporter |
| JY_API_KEY 未配置 | 引导配置 | 请配置 JY_API_KEY，申请邮箱：datamap@gildata.com |
| MCP 服务连接失败 | 建议检查 KEY | 数据源暂时不可用，请检查 JY_API_KEY 是否有效 |
| 工具调用超时 | 重试或降级 | 数据检索超时，正在尝试其他方式 |

## 审查清单

使用前请确认：

- [ ] mcporter 已安装（mcporter --version）
- [ ] MCP 服务已配置（mcporter list 显示两个服务）
- [ ] JY_API_KEY 有效且未过期
- [ ] 时间参数已确认（用户指定或默认）
- [ ] 意图识别准确（宏观/行业/个股/综合）

使用后请确认：

- [ ] 输出无技术细节暴露
- [ ] 所有数据可溯源
- [ ] 时效性审查通过
- [ ] 格式规范统一
- [ ] 无幻觉内容
