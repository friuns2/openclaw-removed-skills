# 投融资事件核验规则

## 关键词判定

### 有效投融资关键词
以下关键词出现时，**可能**为投融资事件：
- 获投、获得投资
- 融资（天使轮、A 轮、B 轮、C 轮、D 轮、Pre-IPO 等）
- 收购、并购
- 入股、参股
- 投资、战略投资
- 加投、追加投资
- 参投、参与投资
- 股权、股权转让、股权收购
- 增资、增资扩股

### 无效情况（不算投融资事件）
以下情况**不能**算作投融资事件：
- **奖励**：政府奖励、补贴、奖金等
- **购买产品**：企业采购产品、设备、服务等
- **减持股权**：股东减持、出售股权
- **贷款**：银行贷款、债权融资（除非明确为股权融资）
- **合作**：单纯的业务合作、战略合作（无股权/资金注入）
- **捐赠**：慈善捐赠、公益捐赠

## 投资方与被投资方判定

### 基本原则
- **被投资方**：获得资金/投资的一方
- **投资方**：提供资金/投资的一方

### 特殊情况处理

#### 股权转让
如"A 受让 B 持有的 C 的股权"：
- **被投资方**：C（标的公司）
- **投资方**：A（受让方）
- **事件描述**：股权收购/股权转让

#### 多方投资
如"A、B 共同投资 C"：
- **被投资方**：C
- **投资方**：A、B
- **事件描述**：联合投资/共同投资

#### 领投与跟投
如"A 领投，B、C 跟投 D 公司"：
- **被投资方**：D
- **投资方**：A（领投）、B（跟投）、C（跟投）
- **事件描述**：X 轮融资（A 领投）

## 投融资轮次识别

### 常见轮次
- 天使轮
- Pre-A 轮
- A 轮
- A+ 轮
- B 轮
- B+ 轮
- C 轮
- D 轮及以后
- Pre-IPO 轮
- 战略融资
- 财务融资

### 金额识别
- 注意金额单位：元、万元、亿元、美元
- 如未明确金额，标注"未披露"

## 被投资方介绍

### 介绍要素
- 公司主营业务
- 所属行业领域
- 核心产品/服务
- 简洁明了（1-2 句）

### 示例
- "专注于动力电池研发"
- "新能源汽车制造商"
- "人工智能芯片设计公司"
- "医疗健康服务提供商"

## 数据源优先级

### 投融资事件获取顺序
1. **IndustryNewsFlash**（核心）- MCP 行业快讯获取投融资事件
2. **NewsInfoList**（补充）- MCP 新闻资讯列表
3. **NewsPublicOpinionList**（补充）- 聚合多家公司新闻舆情

### MCP 调用方式
```bash
mcporter call jy-financedata-api.IndustryNewsFlash query="行业名称"
mcporter call jy-financedata-api.NewsInfoList query="融资 投资"
mcporter call jy-financedata-api.NewsPublicOpinionList query="相关股票代码"
```

### 数据筛选技巧
- 筛选包含关键词的新闻："融资"、"获投"、"收购"、"入股"、"投资"等
- 如 24 小时内数据较少，可放宽至过去一周
- 聚合多家相关公司数据以增加覆盖面

## 配置检查

- [ ] 已连接 MCP 聚源金融数据库
- [ ] `IndustryNewsFlash` 工具可用

## 核验清单

- [ ] 已调用 `IndustryNewsFlash` 获取投融资事件（必需）
- [ ] 数据不足时已调用 `NewsInfoList` 或 `NewsPublicOpinionList` 补充
- [ ] 事件包含有效投融资关键词
- [ ] 排除奖励、购买产品、减持等无效情况
- [ ] 正确识别被投资方和投资方
- [ ] 股权转让情况正确处理（被投资方=标的公司）
- [ ] 投融资轮次标注清晰
- [ ] 被投资方介绍简洁准确
- [ ] 表格格式正确（被投资方 | 投资方 | 投融资事件 | 发布时间）

## 无数据处理

如 24 小时内无投融资事件：
```
### 4. 投融资事件：
无
```

**注意**：无数据时**不要**做成表格，直接写"无"。
