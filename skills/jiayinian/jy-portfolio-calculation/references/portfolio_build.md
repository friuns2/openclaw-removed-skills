# PortfolioBuild - 组合建仓工具

## 功能描述

根据用户提供的资产输入信息，构建一个模拟的投资组合并生成组合 ID。

## 服务器

`jy-financedata-api`

## 调用格式

```bash
mcporter call jy-financedata-api.PortfolioBuild query='<自然语言描述>'
```

## 输入参数

**`query`** (string, 必需) - 自然语言描述，包含：

| 信息 | 说明 | 示例 |
|------|------|------|
| 建仓日期 | YYYY-MM-DD 格式，**不能是当天或未来** | `2026-03-20` |
| 证券信息 | 证券名称 + 代码 + 数量/权重 | `贵州茅台 600519 1000 股，权重 50%` |
| 业绩基准 | 可选 | `沪深 300` |

## 调用示例

### 基础建仓

```bash
mcporter call jy-financedata-api.PortfolioBuild query='2026-03-20 建仓，买入贵州茅台 600519 1000 股，权重 50%，中国平安 601318 2000 股，权重 50%，业绩基准沪深 300'
```

### 返回结果

    "table_markdown": "|组合 ID|调仓建仓结果 | 错误原因|\n|---|---|---|\n|0439d6e585034f46846d0a70b5a967e0|成功||",

## 输出表格

| 字段 | 说明 |
|------|------|
| `simulationCode` | 生成的组合 ID (32 位十六进制) |
| `status` | 操作状态 (成功/失败) |
| `errorMsg` | 错误原因 (如果失败) |

## 权重验证与现金补充

### 规则

- ✅ 权重总和必须等于 1 (100%)
- ⚠️ 如果用户输入的权重总和 **不足 1**，自动用**现金**补充差额
- ❌ 如果权重总和 **超过 1**，拒绝执行并提示用户调整

### 现金补充逻辑

```python
# 伪代码示例
total_weight = sum(position['weight'] for position in positions)

if total_weight > 1:
    raise Error(f"权重总和为{total_weight}，超过 100%，请调整")
elif total_weight < 1:
    cash_weight = 1 - total_weight
    positions.append({
        'symbol': 'CASH',
        'name': '现金',
        'weight': cash_weight,
        'note': '自动补充 (无利息)'
    })
```

### 示例

**用户输入：**
```
"2026-03-20 建仓，买入贵州茅台 600519 权重 50%，中国平安 601318 权重 40%"
```

**自动补充后：**
| 证券 | 权重 |
|------|------|
| 贵州茅台 | 50% |
| 中国平安 | 40% |
| 现金 | 10% ← 自动补充 |

**调用时：**
```bash
mcporter call jy-financedata-api.PortfolioBuild query='2026-03-20 建仓，买入贵州茅台 600519 权重 50%，中国平安 601318 权重 40%，现金 10%'
```

## 日期处理策略

**当用户请求使用当天或未来日期建仓时：**

1. **自动调整**：使用上一个交易日（通常是昨天）执行
2. **明确提醒**：必须告知用户日期已调整
3. **说明原因**：解释服务端限制

**示例：**
```
用户："今天建仓，买入贵州茅台"
→ 回复："⚠️ 系统不允许使用当天日期进行建仓，已自动使用 2026-03-23（上一交易日）的收盘价执行"
```

## 注意事项

1. **日期限制**：建仓日期不能是当天或未来日期，自动使用上一交易日
2. **组合 ID 保存**：建仓成功后必须保存返回的组合 ID，后续所有操作都需要此 ID
3. **组合 ID 格式**：32 位十六进制 (如 `0439d6e585034f46846d0a70b5a967e0`)
4. **现金补充**：权重不足 1 时自动补充现金，现金无利息

## 后续操作

建仓成功后，应**串行**执行以下查询 (不得并行)：

1. **成分券查询** → `PortfolioPositionQuery`
2. **交易流水查询** → `TradeFlowQuery`
3. **绩效指标查询** → `PortfolioIndicatorQuery`

## 错误处理

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| 导入文件中不能填写当天以及未来日期 | 日期是当天或未来 | 使用历史日期 (至少昨天) |
| 证券代码无效 | 代码格式错误或不存在 | 检查代码格式 |
| 权重总和不等于 1 | 用户输入的权重有误 | 提醒用户调整权重 |
