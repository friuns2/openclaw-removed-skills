# PortfolioRebalance - 组合调仓工具

## 功能描述

对已有的模拟组合进行调整，执行证券的资产配置比例的调仓操作。

## 服务器

`jy-financedata-api`

## 调用格式

```bash
mcporter call jy-financedata-api.PortfolioRebalance query='<自然语言描述>'
```

## 输入参数

**`query`** (string, 必需) - 自然语言描述，包含：

| 信息 | 说明 | 示例 |
|------|------|------|
| 组合 ID | 完整的 32 位十六进制组合 ID | `0439d6e585034f46846d0a70b5a967e0` |
| 调仓日期 | YYYY-MM-DD 格式，**不能是当天或未来** | `2026-03-21` |
| 新权重配置 | 各证券的新权重 | `贵州茅台权重 30%，中国平安权重 70%` |

## 调用示例

### 调整权重

```bash
mcporter call jy-financedata-api.PortfolioRebalance query='组合 0439d6e585034f46846d0a70b5a967e0 在 2026-03-21 调仓，贵州茅台权重 30%，中国平安权重 70%'
```

## 输出表格

| 字段 | 说明 |
|------|------|
| `simulationCode` | 组合 ID (与输入一致) |
| `status` | 操作状态 (成功/失败) |
| `errorMsg` | 错误原因 (如果失败) |

## 权重验证与现金补充

### 规则

- ✅ 权重总和必须等于 1 (100%)
- ⚠️ 如果用户输入的权重总和 **不足 1**，自动用**现金**补充差额
- ❌ 如果权重总和 **超过 1**，拒绝执行并提示用户调整

### 现金补充逻辑

```python
# 伪代码示例
total_weight = sum(position['weight'] for position in positions)

if total_weight > 1:
    raise Error(f"权重总和为{total_weight}，超过 100%，请调整")
elif total_weight < 1:
    cash_weight = 1 - total_weight
    positions.append({
        'symbol': 'CASH',
        'name': '现金',
        'weight': cash_weight,
        'note': '自动补充 (无利息)'
    })
```

### 示例

**用户输入：**
```
"组合 0439d6e585034f46846d0a70b5a967e0 在 2026-03-21 调仓，贵州茅台权重 30%，中国平安权重 50%"
```

**自动补充后：**
| 证券 | 权重 |
|------|------|
| 贵州茅台 | 30% |
| 中国平安 | 50% |
| 现金 | 20% ← 自动补充 |

**调用时：**
```bash
mcporter call jy-financedata-api.PortfolioRebalance query='组合 0439d6e585034f46846d0a70b5a967e0 在 2026-03-21 调仓，贵州茅台权重 30%，中国平安权重 50%，现金 20%'
```

## 日期处理策略

**当用户请求使用当天或未来日期调仓时：**

1. **自动调整**：使用上一个交易日（通常是昨天）执行
2. **明确提醒**：必须告知用户日期已调整
3. **说明原因**：解释服务端限制

**示例：**
```
用户："今天调仓，等权配置"
→ 回复："⚠️ 系统不允许使用当天日期进行调仓，已自动使用 2026-03-23（上一交易日）的收盘价执行"
```

**服务端限制：**
恒生聚源服务端代码强制校验调仓日期必须早于当天，否则抛出异常：
> "导入文件中不能填写当天以及未来日期"

## 注意事项

1. **只需 portfolio_id**：不需要额外的 `action` 参数，组合 ID 在自然语言描述中提供
2. **日期限制**：调仓日期不能是当天或未来日期，自动使用上一交易日
3. **串行执行**：调仓后，查询操作必须串行执行，不得并行
4. **现金补充**：权重不足 1 时自动补充现金，现金无利息

## 后续操作

调仓成功后，应**串行**执行以下查询 (不得并行)：

1. **成分券查询** → `PortfolioPositionQuery`
2. **交易流水查询** → `TradeFlowQuery`
3. **绩效指标查询** → `PortfolioIndicatorQuery`

## 错误处理

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| 导入文件中不能填写当天以及未来日期 | 日期是当天或未来 | 使用历史日期 |
| 组合 ID 不存在 | 组合 ID 错误 | 检查组合 ID 格式 (32 位十六进制) |
| 权重总和不等于 1 | 用户输入的权重有误 | 提醒用户调整权重 |
