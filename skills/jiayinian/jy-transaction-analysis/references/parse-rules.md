# 文本解析规则详解

## 输入格式

用户提供的交易流水文本可能来自不同券商的 APP 或交易系统，格式可能不完全一致。本规则用于从各种格式中提取有效交易记录。

---

## 识别表头

常见的表头关键词包括：

| 字段 | 常见表头变体 |
|------|-------------|
| 交易日期 | 交易日期、日期、时间、成交日期、发生日期 |
| 证券代码 | 证券代码、代码、股票代码、标的代码 |
| 证券名称 | 证券名称、名称、股票名称、标的名称 |
| 操作类型 | 操作、买卖方向、业务类别、交易类型 |
| 成交数量 | 成交数量、数量、股数、成交股数 |
| 成交价格 | 成交价格、价格、成交价、成交均价 |
| 成交金额 | 成交金额、金额、交易额、发生金额 |

---

## 操作类型识别

### 有效操作（保留）

| 关键词 | 归类为 |
|--------|-------|
| 买入、买进、建仓、开仓 | 买入 |
| 卖出、卖出、平仓、减仓 | 卖出 |

### 无效操作（忽略）

| 关键词 | 说明 |
|--------|------|
| 资金转入、银行转证券、充值 | 资金存入 |
| 资金转出、证券转银行、提现 | 资金转出 |
| 利息归本、结息 | 利息收入 |
| 股息、红利、分红 | 股息收入 |
| 配股、配售价 | 配股操作 |
| 转债转股、转股 | 转债转换 |
| ETF 申购、ETF 赎回 | ETF 交易 |
| 融资买入、融券卖出、担保品划转 | 融资融券 |
| 佣金、印花税、过户费 | 费用记录 |
| 新股申购、中签缴款 | 新股相关 |

---

## 数据清洗规则

### 1. 日期格式标准化

将各种日期格式统一为 `YYYY-MM-DD`：

```
2026/03/20 → 2026-03-20
20260320 → 2026-03-20
2026-03-20 14:30:00 → 2026-03-20
2026 年 03 月 20 日 → 2026-03-20
```

### 2. 数字格式处理

- 去除千分位逗号：`1,000,000` → `1000000`
- 处理科学计数法：`1.5E+06` → `1500000`
- 统一小数位数：价格和金额保留 2 位小数

### 3. 证券代码处理

- 去除前后缀：`SH600000` → `600000`，`SZ000001` → `000001`
- 补齐位数：A 股代码统一为 6 位

### 4. 异常数据处理

- 成交数量为 0 或负数 → 忽略该条记录
- 成交价格为 0 或负数 → 忽略该条记录
- 证券代码为空 → 忽略该条记录
- 操作类型无法识别 → 忽略该条记录

---

## 解析流程

```
1. 读取用户输入的文本
   ↓
2. 按行分割，识别表头行
   ↓
3. 根据表头确定各列的索引位置
   ↓
4. 遍历数据行，提取各字段值
   ↓
5. 标准化日期、数字格式
   ↓
6. 过滤无效操作类型
   ↓
7. 过滤异常数据
   ↓
8. 输出结构化交易记录列表
```

```
交易日期    证券代码    证券名称    操作    成交数量    成交价格    成交金额
```

## 边界情况处理

### 1. 多行表头

某些券商的流水可能有多行表头：

```
交易日期    证券代码    证券名称    操作    成交数量    成交价格    成交金额
Date       Code       Name       Type    Quantity    Price      Amount
```

处理：识别包含中文字段的行作为主表头。

### 2. 无表头数据

某些导出的 CSV 可能无表头：

```
2026-03-20,600000,浦发银行，买入，10000,8.50,85000
```

处理：按固定列顺序解析（日期、代码、名称、操作、数量、价格、金额）。

### 3. 混合分隔符

某些文本可能混合使用空格、制表符、逗号：

```
2026-03-20  600000,浦发银行  买入，10000  8.50,85000
```

处理：智能识别分隔符，优先使用制表符/多空格，其次逗号。
