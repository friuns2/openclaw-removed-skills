## 🎯 重构目标

**核心理念转变**：从"固定轮次 + 固定专家" → "需求驱动 + 动态匹配"

### V1 的问题

| 问题 | 说明 | 影响 |
|------|------|------|
| ❌ **需求理解缺失** | 直接开始讨论，不理解用户需求 | 专家匹配不准确 |
| ❌ **专家固定** | 固定 3-5 个专家（工程/设计/测试） | 测试专家参与设计阶段（不合适） |
| ❌ **轮次固定** | 固定 5 轮讨论 | 简单需求也需 5 轮（过度） |
| ❌ **讨论跑题** | 专家被要求讨论不擅长领域 | 输出质量下降 |
| ❌ **成本过高** | 所有需求都用完整 5 轮 | Token 浪费严重 |

### V2 的解决方案

| 解决方案 | 说明 | 收益 |
|---------|------|------|
| ✅ **需求智能拆解** | 先分析需求类型，再匹配专家 | 专家匹配准确率提升 |
| ✅ **动态专家匹配** | 根据需求类型动态选择专家 | 排除不相关专家 |
| ✅ **按议题分治** | 不再固定轮次，按议题讨论 | 讨论更聚焦 |
| ✅ **复杂度适配** | 高/中/低复杂度自动适配 | 成本降低 50% |

## 🔄 流程对比

### V1 流程（固定 5 轮）

```
R1: 独立方案（3 专家各写各的）
  ↓
R2: 相互引用（为了引用而引用）
  ↓
R3: 方案优化（可能偏离重点）
  ↓
R4: 共识形成（可能强行共识）
  ↓
R5: 最终总结（汇总）
```

**问题**：
- 测试专家在设计阶段说"要写测试"（废话）
- 专家被要求讨论自己不擅长的领域
- 轮次固定，简单需求也要 5 轮

### V2 流程（需求驱动）

```
Step 1: 需求智能拆解
  ↓
Step 2: 专家精准匹配
  ↓
Step 3: 用户确认配置
  ↓
Step 4: 按议题分治讨论
  ├─ 议题 1: 技术架构（工程专家主导）
  ├─ 议题 2: AI 功能（AI 工程师主导）
  └─ 议题 3: 用户体验（UX 设计师主导）
  ↓
Step 5: 整合方案
```

**优势**：
- ✅ 每个专家只讨论自己擅长的
- ✅ 讨论有焦点，不跑题
- ✅ 轮次按需，不固定
- ✅ 测试专家不参与设计阶段

## 💰 成本对比

### 场景分布假设

```
高复杂度：20%（核心产品架构）
中复杂度：50%（模块设计、功能规划）
低复杂度：30%（简单功能、代码审查）
```

### V1 vs V2 成本

| 场景 | V1 Token | V2 Token | 节省 |
|------|---------|---------|------|
| **高复杂度** | 80,000 | 80,000 | 0% |
| **中复杂度** | 80,000 | 40,000 | 50% |
| **低复杂度** | 80,000 | 20,000 | 75% |

**加权平均节省**：
```
20%×0% + 50%×50% + 30%×75% = 0 + 25 + 22.5 = 47.5%
```

**实际节省**：约 **50%** 🎉

## 🚀 使用示例

### 基础用法

```bash
# 自动复杂度
RoundTable 讨论一下：智能待办应用的架构设计

# 指定复杂度
RoundTable 高复杂度：智能待办应用从 0 到 1 完整设计
RoundTable 中复杂度：用户认证模块设计
RoundTable 低复杂度：PR 代码审查

# 指定专家
RoundTable 指定专家 [engineering, ux_designer]：任务管理界面设计
```

### Python API

```python
# 分析需求
from roundtable_engine_v2 import analyze_requirement

result = analyze_requirement("智能待办应用的架构设计")
print(result)
# {
#     "topic": "智能待办应用的架构设计",
#     "detected_types": ["architecture", "ai_ml"],
#     "recommended_experts": ["engineering", "ai_engineer"],
#     "excluded_experts": [],
#     "key_topics": [...]
# }

# 运行 RoundTable
from roundtable_engine_v2 import run_roundtable_v2

await run_roundtable_v2(
    topic="智能待办应用的架构设计",
    complexity="auto",
    user_channel="user_channel_id"
)
```
