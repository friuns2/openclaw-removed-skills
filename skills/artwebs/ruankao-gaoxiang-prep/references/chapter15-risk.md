# 第15章 风险管理

## 背诵重点

1. 风险管理七个过程：规划风险管理、识别风险、实施定性风险分析、实施定量风险分析、规划风险应对、实施风险应对、监督风险
2. 风险定义：一种不确定的事件或条件，一旦发生，会对项目目标产生某种正面或负面的影响；项目风险既包括对项目目标的威胁，也包括促进项目目标的机会；项目风险管理旨在利用或强化正面风险（机会）、规避或减轻负面风险（威胁）
3. 已知风险vs未知风险：已知风险是已经经过识别和分析的风险，对其规划应对方案是可行的；未知风险是无法管理的，项目经理采取一般的应急措施处理
4. 风险三种分类（可预测性）：已知风险（分析可知，经常发生且后果轻微）、可预测风险（经验可知，后果难料可能严重）、不可预测风险（无法预见，外部因素天灾人祸）
5. 风险五种分类（后果）：纯粹风险（不能带来机会、无获得利益可能）、投机风险（既可能带来机会、获得利益，又隐含威胁、造成损失）、内部风险vs外部风险
6. 风险三种分类（来源）：自然风险（自然力作用）、人为风险（人的活动带来）
7. 风险三种分类（承担者）：有项目业主风险、政府风险、承包商风险、投资方风险、设计单位风险、监理单位风险、供应商风险、担保方风险和保险公司风险
8. 风险成本包括：有形成本（直接损失如失火重置成本、受伤人员医疗费用、休养费、工资；间接损失如灭火、停工等发生的成本）、无形成本（风险损失减少了机会、风险阻碍了生产率的提高、风险造成资源分配不当）、预防与控制风险的成本
9. 风险登记册内容包括：已识别风险的清单、潜在风险责任人、潜在风险应对措施清单、风险的根本原因、风险更新的时间、风险分类和优先级排序
10. 风险应对策略：威胁应对策略（规避、转移、减轻、上报、接受）、机会应对策略（开拓、分享、提高、接受）、应急应对策略、整体项目风险应对策略

## 补充说明

**风险三种类型（可预测性）：**
- 已知风险：分析可知，经常发生且后果轻微（内部管理为主），应对策略为提前制定标准流程
- 可预测风险：经验可知，后果难料可能严重（履约/技术为主），应对策略为预留应急预案和缓冲
- 不可预测风险：无法预见，外部因素天灾人祸（不可抗力/宏观环境），应对策略为买保险、留现金储备

**风险五种类型（后果）：**
- 纯粹风险：不能带来机会、无获得利益可能（如机房在施工过程中失火）
- 投机风险：既可能带来机会、获得利益，又隐含威胁、造成损失（如投资的房地产开发项目如果失败）
- 风险不是零和游戏：很多情况下，涉及风险的各个方面都要蒙受损失，无一幸免或损失产生

**风险五种类型（来源）：**
- 自然风险：自然力的作用，造成财产毁损或人员伤亡的风险（如发生洪水或地震而造成的工程损害、材料和器材损失）
- 人为风险：由于人的活动而带来的风险（行为、经济、技术、政策和组织风险）

**风险五种类型（承担者）：**
- 有项目业主风险、政府风险、承包商风险、投资方风险、设计单位风险、监理单位风险、供应商风险、担保方风险和保险公司风险

**风险应对策略：**
1. 规避：改变项目计划以消除风险威胁，或保护项目目标不受影响（如采用不太复杂的流程、进行更多测试、选择稳定的供应商）
2. 转移：将风险威胁的影响和所有权转移给第三方（如购买保险、使用成本加成合同、履约保函、担保书、保证书）
3. 减轻：降低风险的概率和/或影响（如采用更简单的流程、进行更多测试、使用原型、选择更可靠的供应商）
4. 上报：项目团队无法自行应对，风险威胁超出其权限或资源范围
5. 接受：接受威胁，不采取行动（如主动接受：建立应急储备；被动接受：不采取任何行动，直到风险发生）
6. 开拓：确保机会肯定发生（如分配最有能力的资源到项目以缩短时间）
7. 提高：提高机会发生的概率和/或影响（如为项目分配更多资源）
8. 分享：将机会的所有权或部分所有权分配给第三方（如建立合作联盟、合资企业、特殊目的公司）
9. 应急应对策略：针对已知的未知风险（可预测风险）而制定的应对措施
10. 整体项目风险应对策略：面对整体项目风险，可以采用规避、开拓、减轻或转移策略，或者接受部分或全部影响，并准备相应的应对措施

**风险识别工具与技术：**
1. 文档审查：对项目文档进行结构化审查，识别项目假设与风险之间的不一致
2. 信息收集技术：头脑风暴、德尔菲技术、访谈、根本原因分析
3. 核对单：基于历史信息和类似项目的知识编制的清单
4. 假设分析：检验假设条件在项目中的有效性，并识别由此引发的风险
5. SWOT分析：优势、劣势、机会和威胁
6. 专家判断

**实施定性风险分析工具与技术：**
1. 风险概率和影响评估：评估风险发生的概率和影响程度
2. 概率和影响矩阵：将风险按概率和影响程度进行分类
3. 风险数据质量评估：评估风险数据的准确性、可靠性、完整性
4. 风险分类：按风险来源、风险类型、风险影响等分类
5. 风险紧迫性评估：评估风险的紧急程度
6. 专家判断

**实施定量风险分析工具与技术：**
1. 数据收集：访谈、概率分布
2. 不确定性表现方式：概率分析、敏感性分析、决策树分析和影响图
3. 模拟：蒙特卡洛分析
4. 专家判断

**规划风险应对工具与技术：**
1. 专家判断
2. 人际关系与团队技能
3. 数据收集：访谈、根本原因分析
4. 数据表现：概率和影响矩阵、层级图、决策树
5. 决策：多标准决策分析

**风险登记册更新：**
在规划风险应对过程中，风险登记册需要更新以下内容：
1. 商定的应对策略和措施
2. 实施选择的应对策略所需的具体行动
3. 风险触发条件和预警信号
4. 风险责任人和时间表
5. 应急储备和应急计划
6. 弹性措施

**监督风险工具与技术：**
1. 数据分析：技术绩效分析、储备分析、审计
2. 风险审查：定期审查风险登记册
3. 会议：风险审查会议
4. 项目管理信息系统

**风险分类记忆口诀：**
- 按后果：纯投外（纯粹、投机、外部）
- 按来源：自人（自然、人为）
- 按承担者：业政承投设监保（业主、政府、承包、投资、设计、监理、担保、保险）
- 按可预测性：已可不可（已知、可预测、不可预测）

**风险管理新实践：**
1. 非事件类风险：变异性风险、模糊性风险
2. 项目韧性：应对突发性风险
3. 整合式风险管理：跨部门协作

**风险成本：**
- 有形成本：直接损失+间接损失
- 无形成本：风险损失减少了机会+风险阻碍了生产率的提高+风险造成资源分配不当
- 预防与控制风险成本：向保险公司投保、向有关方面咨询、配备必要的人员、购置用于预防和减损的设备、对有关人员进行教育或训练以及人和设备的维持和维护费用

**风险优先级排序：**
- 概率高且影响大的风险优先级最高
- 概率低且影响小的风险优先级最低
- 使用概率和影响矩阵来确定风险的优先级

**风险管理的目的：**
- 降低风险不利影响
- 提高项目成功的可能性
