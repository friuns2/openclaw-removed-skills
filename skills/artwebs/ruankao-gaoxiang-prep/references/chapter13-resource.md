# 第13章 资源管理

## 背诵重点

1. 资源管理六个过程：规划资源管理、估算活动资源、获取资源、建设团队、管理团队、控制资源
2. 项目资源包括团队资源（人力资源）和实物资源（设备、材料、设施和基础设施），实物资源指人力资源以外的任何资源
3. 项目经理的5种权力来源：职位权力（来自组织的正式授权）、惩罚权力（负面手段，谨慎使用）、奖励权力（给予奖励的能力）、专家权力（专业技能）、参照权力（学习榜样的人格魅力）；职位/惩罚/奖励权力来自组织授权，专家/参照权力来自管理者自身
4. 优秀团队建设5个阶段：形成阶段（相互认识）、震荡阶段（冲突矛盾）、规范阶段（协同工作相互信任）、发挥阶段（组织有序平稳高效）、解散阶段（释放人员解散团队）；不管处于什么阶段，增加或减少一个人，都从形成期重新开始
5. 马斯洛需求层次理论5层：生理需求（衣食住行）、安全需求（人身安全生活稳定）、社会交往需求（友谊爱情）、受尊重需求（自尊心和荣誉感）、自我实现需求（发挥个人能力）
6. 赫茨伯格双因素理论：保健因素（工作环境、工资薪水、公司政策等，不满足会不满意，满足了不会更满意）、激励因素（成就、承认、工作本身、责任、发展机会，不满足不会不满意，满足了才会更满意）
7. X理论（不好）vs Y理论（好）：X理论认为人天生好逸恶劳、以自我为中心、缺乏进取心，前期用X理论指导管理；Y理论认为人天生热爱工作、能够自我控制，中期执行时用Y理论授权团队
8. 期望理论：激励水平=目标效价×期望值；目标效价（实现该目标对个人有多大价值）、期望值（个人对实现该目标可能性大小的主观估计）
9. 领导者vs管理者：领导者主要工作是确定方向、统一思想、激励和鼓舞；管理者负责召集人员、分配任务；领导者设定目标，管理者率众实现目标；项目经理具有领导者和管理者的双重身份
10. RACI矩阵：执行（Responsible）、负责（Accountable）、咨询（Consulted）、知会；职责分配矩阵RAM显示了分配给每个工作包的项目资源，说明工作包或活动与项目团队成员之间的关系

## 补充说明

**RACI模型：**
- R（Responsible/执行）：实际执行任务的人
- A（Accountable/负责）：对任务负最终责任，有签字批准权
- C（Consulted/咨询）：在决策前需要咨询的人
- I（Informed/知情）：需要被告知结果的人

**资源管理计划内容：**
- 识别资源：识别和量化项目所需的团队和实物资源的方法
- 获取资源：如何获取项目所需的团队和实物资源的指南
- 角色与职责：角色（职务或分配给某人的职务）、职权（使用项目资源、做出决策、签字批准的权力）、职责（必须履行的工作）、能力（必须具备的技能和才干）
- 项目组织图：以图形方式展示项目团队成员及其报告关系
- 项目团队资源管理：何定义、配备、管理和最终遣散项目团队资源的指南
- 培训：针对项目成员的培训策略
- 团队建设：建设项目团队的方法
- 资源控制：依据需要确保实物资源充足可用的方法
- 认可计划：将给予团队成员哪些认可和奖励，以及何时给予

**团队章程内容：团队价值观、沟通指南、决策标准和过程、冲突处理过程、会议指南、团队共识【口诀：团购绝对充值】**

**数据表现类型：**
1. 层级型：可采用传统的组织结构图（OBS），自上而下地显示各种职位及其相互关系；工作分解结构（WBS）用来显示如何把项目可交付成果分解为工作包
2. 矩阵型：职责分配矩阵（RAM），显示了分配给每个工作包的项目资源；RACI矩阵是RAM的一个例子
3. 文本型：如果需要详细描述团队成员的职责，就可以采用文本型，提供诸如职责、职权、能力和资格等方面的信息

**资源分解结构RBS：**
按资源类别和类型，对团队和实物资源的层级列表，用于规划、管理和控制项目工作；资源类别包括（但不限于）人力、材料、设备和用品；资源类型则包括技能水平、要求证书、等级水平

**获取资源工具与技术：**
1. 决策：多标准决策分析，使用标准包括可用性、成本、能力、经验、知识、技能、态度、国际因素
2. 人际关系与团队技能：谈判（与职能经理、其他项目管理团队、外部组织和供应商）
3. 预分派：事先确定项目的实物或团队资源（在竞标过程中承诺分派、项目取决于特定人员的专有技能、制定项目章程过程中已指定某些团队成员）
4. 虚拟团队：具有共同目标，在完成角色任务的过程中很少或没有时间面对面工作的一群人

**获取资源输出：**
1. 物质资源分配单：记录了项目将使用的材料、设备、用品、地点和其他实物资源
2. 项目团队派工单：记录了团队成员及其在项目中的角色和职责，可包括项目团队名录
3. 资源日历：识别每种具体资源可用时的工作日、班次、正常营业的上下班时间、周末和公共假期

**建设团队工具与技术：**
1. 集中办公：把许多或全部最活跃的项目团队成员安排在同一个地点工作，以增强团队工作能力
2. 虚拟团队：使用更多技术熟练的资源、降低成本、减少出差及搬迁费用
3. 沟通技术：共享门户、视频会议、音频会议、电子邮件/聊天软件
4. 人际关系与团队技能：冲突管理、影响力、激励、谈判、团队建设
5. 认可与奖励：满足被奖励者重要需求的奖励才是有效的
6. 培训：提高项目团队成员能力，可以是正式或非正式的
7. 个人和团队评估：让项目经理和项目团队洞察成员的优势和劣势

**建设团队目标：**
1. 提高团队成员的知识和技能
2. 提高团队成员之间的信任和认同感
3. 创建富有生气、凝聚力和协作性的团队文化
4. 提高团队参与决策的能力

**团队绩效评价指标：**
1. 个人技能的改进
2. 团队能力的改进
3. 团队成员离职率的降低
4. 团队凝聚力的加强

**管理团队工具与技术：**
1. 人际关系与团队技能：冲突管理、制定决策、情商、影响力、领导力
2. 项目管理信息系统

**控制资源工具与技术：**
1. 数据分析：备选方案分析、成本效益分析、绩效审查、趋势分析
2. 问题解决
3. 人际关系与团队技能：谈判、影响力
4. 项目管理信息系统

**领导者vs管理者：**
- 领导者工作：确定方向、统一思想、激励和鼓舞
- 管理者工作：召集人员、分配任务、实现目标
- 领导者设定目标，管理者率众实现目标

**激励理论对比：**
- 马斯洛需求层次：5层从低到高（生理→安全→社会交往→受尊重→自我实现）
- 赫茨伯格双因素：保健因素（防止不满意）、激励因素（产生满意感）
- X理论：人天生好逸恶劳，前期用
- Y理论：人天生热爱工作，中期执行时用
- 期望理论：M=V×E（激励=目标效价×期望值）
