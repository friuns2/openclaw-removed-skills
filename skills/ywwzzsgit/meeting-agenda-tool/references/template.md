# 会议议程模板规范

## 字体规范

| 元素 | 字体 | 字号 | 样式 |
|------|------|------|------|
| 会议名称（文档标题） | 仿宋 GB2312 | 小一（24pt） | 加粗、居中 |
| 一级标题（环节序号+名称 / 人员） | 黑体 | 三号（16pt） | 加粗、左对齐 |
| 字段标签（时间、地点、内容、议程、主持人等） | 黑体 | 三号（16pt） | 加粗 |
| 正文内容（字段值、议程项目） | 仿宋 GB2312 | 三号（16pt） | 正常 |
| 参会单位名称 | 楷体 GB2312 | 三号（16pt） | 加粗，前空两个全角空格 |
| 参会成员 | 仿宋 GB2312 | 三号（16pt） | 正常，姓名与职务之间空两个全角空格，不加"-" |

## Markdown 模板结构

```markdown
# [会议/活动名称]

## 一、[环节名称，如：产品演示]

- **时间**：[日期] [星期] [开始时间]-[结束时间]
- **地点**：[具体地点]
- **内容**：[简要描述]

## 二、[环节名称，如：座谈交流]

- **时间**：[日期] [星期] [开始时间]-[结束时间]
- **地点**：[具体地点]
- **主持人**：[姓名]（可选）
- **议程**：
  1. [议程项1]
  2. [议程项2]
  3. [议程项3]
- **人员**：

### [参会单位A]

- [姓名]　　[职务]
- [姓名]　　[职务]
- [姓名]　　[职务]  
  [补充说明，如分管部门]

### [参会单位B]

- [姓名]　　[职务]
- [姓名]　　[职务]
```

## JSON 数据格式（供脚本使用）

```json
{
  "title": "XXX座谈交流会议议程",
  "filename": "XXX座谈交流会议议程",
  "sections": [
    {
      "name": "产品演示",
      "time": "2026年4月2日（星期四）09:00-10:00",
      "location": "XX楼XX会议室",
      "content": "演示产品核心功能及技术方案"
    },
    {
      "name": "座谈交流",
      "time": "2026年4月2日（星期四）10:00-12:00",
      "location": "XX楼XX会议室",
      "host": "张三",
      "agenda": [
        "介绍合作背景与目标",
        "各方发言交流",
        "讨论下一步合作计划"
      ],
      "persons": [
        {
          "unit": "XX公司",
          "members": [
            {"name": "李四", "title": "总经理"},
            {"name": "王五", "title": "技术总监", "note": "分管研发部"}
          ]
        },
        {
          "unit": "YY单位",
          "members": [
            {"name": "赵六", "title": "处长"},
            {"name": "钱七", "title": "科长"}
          ]
        }
      ]
    }
  ]
}
```

## 注意事项

1. `filename` 字段为输出文件名（不含扩展名），建议使用会议名称
2. 环节序号自动按"一二三四..."排列，无需手动填写
3. `persons` 中每个单位为一组，`members` 中每个成员可选填 `note` 作为补充说明
4. 时间格式建议统一为：`YYYY年M月D日（星期X）HH:MM-HH:MM`
