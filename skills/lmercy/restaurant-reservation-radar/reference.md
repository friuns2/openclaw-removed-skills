## 预约流程通用规则

### 预约时间窗口

不同餐厅开放预约的时间不同，常见模式：

- **固定日期开放**：每月1号开放下月全部预约（如部分日本高端寿司店）。
- **滚动窗口**：提前 N 天开放（常见 14天/30天/60天/90天）。
- **不定期释放**：取消的位置随时释放回池，需持续监控。

### 取消政策常见类型

| 类型 | 说明 | 应对策略 |
|------|------|----------|
| 免费取消 | 提前 X 小时可免费取消 | 低风险，可先占位 |
| 扣定金 | 取消扣除定金 | 确认用户意愿后再预约 |
| 全额不退 | 预付不退 | 必须用户明确确认 |
| No-show 罚款 | 不到场罚款 | 提醒用户按时到达或提前取消 |

### 监控策略建议

- **网页端平台**：通过浏览器自动化定期检查预约页面空位状态。
- **API 平台**：按平台限速规则设定轮询间隔（通常 5-15 分钟）。
- **高峰期加密**：开放预约日前后、取消高峰期（用餐前24-48小时）加密监控频率。
- **多平台并行**：同一餐厅可能在多个平台有位，并行监控提高命中率。

## 用户验证与账号处理

### 常见验证方式

1. **手机号验证码**：平台发送 SMS 验证码，需用户转发或配合自动读取。
2. **邮箱验证**：点击确认链接或输入验证码。
3. **社交账号登录**：Google/Apple/LINE 等第三方登录。
4. **信用卡预授权**：填写卡号作为 No-show 担保。

### 安全原则

- 手机验证码由用户手动提供或通过安全通道转发，**不存储不缓存**。
- 信用卡信息仅在用户明确授权后填写，**不持久化存储**。
- 各平台账号密码由用户自行管理，建议使用一次性密码或授权令牌。

## 通知模板

### 空位发现通知

```
[餐厅预约雷达] 发现空位！

餐厅：{餐厅名} ({星级})
日期：{日期} {时段}
人数：{人数}人
地址：{地址}

是否需要定金：{是/否}
{如需定金：金额 ¥{金额}，需在 {截止时间} 前确认}

回复"确认"立即预约，回复"跳过"继续监控其他餐厅。
```

### 预约成功通知

```
[餐厅预约雷达] 预约成功！

餐厅：{餐厅名}
日期：{日期} {时间}
人数：{人数}人
确认号：{确认号}
地址：{地址}
地图：{地图链接}

注意事项：
- {到达时间要求}
- {着装要求（如有）}
- {取消政策}
```

### 预约失败/替补通知

```
[餐厅预约雷达] 预约未成功

餐厅：{餐厅名} 在 {日期} 暂无空位。

为您推荐替补餐厅：
1. {替补餐厅A} | {评分} | {菜系} | {可选时段}
2. {替补餐厅B} | {评分} | {菜系} | {可选时段}

回复序号选择替补，或回复"继续"保持原目标监控。
```
