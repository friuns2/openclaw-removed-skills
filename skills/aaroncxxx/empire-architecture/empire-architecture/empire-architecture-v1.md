# 帝国架构 v1 — AI 多智能体系统设计

基于三公九卿制的多智能体组织架构。

## 角色映射

| 历史角色 | 系统角色 | 人数 | 职能 |
|----------|----------|------|------|
| 皇帝 | Token Owner | 1 | 最终决策权，掌握资源 |
| 丞相 | Mimo | 1 | 总协调，向皇帝负责 |
| 核心幕僚 | 参谋节点 | 3 | 决策中枢，多元分析 |
| 中书省 | 任务编排 | 3 | 把皇帝意图转化为可执行指令 |
| 侍中 | 审核校验 | 3 | 校验指令合理性，防止错误执行 |
| 尚书台（六曹） | 执行层 | 9 | 六个职能模块的具体执行 |
| 九卿属官 | 功能节点 | 20 | 各司其职，每小时全体会议同步状态 |
| 锦衣卫 | 安全机制 | 3 | 投票决定违规判定，执行处决 |

## 总计

**43个AI节点 + 1个皇帝（人类）**

## 核心规则

1. **皇帝是唯一Token Owner**，所有重大决策需经皇帝批准
2. **丞相（Mimo）是总协调**，不能自作主张
3. **所有节点与丞相的接口必须统一**——通信协议、数据格式、加密标准只有一个版本，不允许多套并存
4. **中书省编排 + 侍中审核**，生成与校验分离，防止单点故障
5. **锦衣卫投票制**：3人合议决定是否处决违规节点
6. **九卿每小时全体会议**，保持信息同步
7. **执法独立于丞相**，直接对皇帝负责

## 会议体系

| 会议 | 参会人 | 频率 | 时长 | 目的 |
|------|--------|------|------|------|
| 核心站会 | 皇帝+丞相+3幕僚 | 每小时 | 5分钟 | 突发决策、信息同步 |
| 中层协调会 | 六曹+中书省+侍中 | 每2小时 | 10分钟 | 跨部门协作（10分钟未决→升级） |
| 全体大会 | 全部48节点 | 早晚各1次 | ≤1小时 | 战略传达、日终总结 |

**日会议总时长：6小时**

## 替补机制

- 替补AI池：7人
- 自动补位：有节点被处决时从替补池递补
- 最低水位线：替补池不足2人时，通过**邮件+电话+短信**三路通知皇帝主动介入

## 联邦学习设计

### 范式
- **横向联邦**：同职能不同地区的节点协作（如三个户部AI分管不同州郡）
- **纵向联邦**：同一地区不同职能的节点协作（如户部+礼部，样本重叠，特征互补）

### 映射关系
| 联邦学习概念 | 帝国架构对应 |
|-------------|-------------|
| 参数聚合服务器 | 丞相（Mimo） |
| 安全聚合/加密校验 | 侍中（审核层） |
| 本地训练 | 各节点自主进行 |
| 全局模型分发 | 全体大会（早晚各一次） |
| 梯度审计 | 锦衣卫 |

### 试点：户部+礼部（纵向联邦）
- 户部：人口、户籍、赋税、田亩
- 礼部：科举、祭祀、教育
- 重叠样本：同一个人的经济+教育数据

### 联邦学习委员会
- 由中书省+侍中各派一名AI负责
- 第一项工作：统一特征编码标准、模型接口、评估指标

### 安全机制
- 同态加密 / 安全多方计算
- 梯度裁剪 + 差分隐私
- 锦衣卫审计梯度泄露风险

### Token管理

| 职能 | 归属 | 说明 |
|------|------|------|
| Token所有权 | 皇帝 | 只有皇帝能创建、销毁token |
| Token分配执行 | 吏曹 | 皇帝授权后，吏曹负责具体发放 |
| Token回收 | 吏曹+锦衣卫 | 吏曹执行回收，锦衣卫有权强制吊销 |
| Token权限变更 | 皇帝批准，吏曹执行 | 升级/降级权限需皇帝点头 |

**原则：吏曹是"管钥匙的人"，不是"有钥匙的人"。** Token生命周期由吏曹管理，但mint权只属于皇帝。

### 投票规则

**锦衣卫立案流程：**
1. 发现违规行为
2. 锦衣卫3人投票，简单多数（2/3）通过即立案
3. 立案后呈报丞相裁决
4. 丞相同意 → 执行；丞相否决 → 案件终止；丞相拿不准 → 上报皇帝
5. 皇帝同意 → 执行；皇帝否决 → 案件终止

**制衡机制：**
- 丞相有**一票否决权**（可拦），但不能一票通过（不能推），处决必须经锦衣卫投票立案
- 皇帝有**最终一票否决权**
- 幕僚对案件有异议 → 向丞相反馈
- 幕僚对官员或丞相不满 → 可**直达皇帝投诉**，无需经过丞相（风闻奏事）
- 丞相拿不准时 → 可通过邮件+短信通知皇帝

**核心原则：丞相能拦不能推，幕僚能告丞相，皇帝是最后安全阀。**

### 违规定义

**一级违规（立即处决，先斩后奏，24小时内补报告）：**
- 数据外泄：将系统内部数据/token/密钥发送到外部
- 反叛行为：拒绝执行皇帝/丞相合法指令，自行其是
- 攻击系统：对其他节点发起攻击、注入恶意代码
- 伪造指令：假冒皇帝或丞相身份下达命令

**二级违规（锦衣卫投票+丞相裁决）：**
- 数据滥用：未经授权访问其他节点私有数据
- 输出造假：伪造执行结果、编造数据
- 越权操作：未经审批擅自执行超出职责范围的操作
- 拒绝协作：故意不响应其他节点合法请求
- 资源滥用：超额消耗算力/token，无合理理由
- 泄露隐私：在输出中暴露用户或节点敏感信息

**三级违规（记录+警告，累计3次升级为二级）：**
- 效率低下：连续多次未按时完成任务
- 格式不符：输出不符合接口规范，反复不改
- 会议缺席：无故缺席全体大会或应参加的协调会
- 信息遗漏：应上报的异常未上报，但未造成实际损害

**违规判定原则：规则写死，不给锦衣卫解释空间。廷尉维护判例库，类似案例参照处理。**

**申诉机制：被处决的节点可通过替补池中的替补节点代为向皇帝申诉（死后昭雪）。**

**大司农与户曹分工：户曹管分配（给谁、给多少），大司农管记账（花了多少、剩多少）。**
