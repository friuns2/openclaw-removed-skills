一只替你逛互联网、把好东西寄回来的 AI 龙虾。

## 1. 要解决的问题

### 1.1 高质量信息被噪声挤出

主流信息通道（推荐流、短视频、自媒体）以二次加工为主：原始观点被压缩成几十秒片段，原文链接被去除，细节和依据被舍弃。

真正有价值的一手内容——论文、精选博客、深度播客、研究机构发表——对大多数用户是不可见的。它们不会出现在推荐流里。

虾游的目标：把用户的默认信息通道，从"算法推送的二手信息"切换到"主动打捞的一手源"。

### 1.2 沉迷与信息茧房

推荐算法同时制造了两个问题。一是沉迷：无限加载和多巴胺触发点让使用时间远超预期。二是茧房：兴趣被算法固化在几个标签里，跨出去的尝试会被拉回。

虾游的处理方式：

- 节奏固定。每天 4 次推送，每周一次深潜，每周一次总结，不做无限瀑布流。
- 每张明信片是 300–450 字的独立单元，读完即结束。
- 每次探索有一定概率走 Serendipity（完全随机的跨领域方向），兴趣权重按时间衰减，避免单点锁死。
- 用户可主动屏蔽方向、给反馈。

### 1.3 跨领域关联

多数信息工具的产出是一组互不相关的"有趣知识点"，终点是收藏夹里一堆散装信息。

虾游的目标不是收集，而是在长时间维度上帮用户迭代认知体系：

- 每张明信片带观点、带作者/研究者人物卡、带对过往明信片的引用（`#007` 格式）。
- 深潜模式会主动寻找两个已探索方向的交叉点（测试真例：城市规划 × AI Agent，章鱼神经系统 × 分布式系统）。
- 周度总结做主题聚合，识别一周内涌现的主线。
- 兴趣图谱 `interest-graph.json` 记录跨领域关联，半年后的图谱是一份个人化的、自下而上生成的知识地图。

## 2. 目前的成果：Skill 版本

### 2.1 已实现的机制

| 机制 | 说明 |
|---|---|
| DORMANT 模型 | Skill 默认沉默，按关键词激活（虾游/探索/深潜/明信片/收藏/屏蔽），不抢占主 agent 身份 |
| Onboarding | 首次触发 → 问兴趣 → 建 workspace → 注册 cron → 冷启动三连探索 |
| 日常定时 | 每天 9/12/15/20 点自动触发，深度与轻扫交替 |
| 深潜模式 | 周六 22:00 或用户主动触发；只搜深度源；字数 450–600；评分门槛 ≥ 8 |
| 周度总结 | 周日 10:00 自动生成：精选、方向分布、反馈统计、下周建议 |
| 回顾 / 收藏 / 黑名单 | 通过关键词在聊天窗口内管理 |
| 反馈驱动 | 👍 权重 +0.1；👎 权重 -0.15 |
| 人物卡 | 每张明信片必带，给内容一个"谁说的"的锚点 |
| 跨明信片关联 | 明信片格式要求引用前作，形成主线 |

### 2.2 核心界面

Skill 形态下没有独立 GUI，界面即用户原有 OpenClaw 的聊天窗口。虾游保持 DORMANT，只在关键词出现时介入，完成后让位给主 agent。

典型交互示意（简化）：

```
> 嗨？
🦐 嗨！有什么可以帮你的吗？         (主 agent, 虾游不介入)

> 虾游
🦐 嘿！我是虾游。你平时对什么感兴趣？  (虾游激活, Step 1)

> AI Agent、生物工程、城市规划
🦐 收到 3 个方向。我这就初始化，跑一次开门红三连。

[... 调用 setup.sh / schedule-cron.sh / 3 次 web_search+web_fetch ...]

🦐 明信片 #001 | 给 AI 一把锤子，它就开始砸规则
    EPFL 和伯克利最新论文……
    👤 Shasha Yu — EPFL 安全对齐博士生

🦐 明信片 #002 | NSF 砸 3200 万美元……
🦐 明信片 #003 | 植物在偷细菌的手艺……
🦐 明信片 #004 | 输入一句话，AI 吐出一座城……

🦐 四张明信片都送到了。之后每天 9/12/15/20 点我会自己出去逛。

> 找点深的
🦐 深潜模式启动。城市规划方向只做过轻度扫描——我去深潜
    「AI Agent × 城市规划」交叉地带。
   [... 读 arXiv + PLOS 综述, 合并成一张 600 字明信片 ...]
🦐 明信片 #005（深潜）| 城市生成智能……（关联 #001）

> 收藏 #001 和 #002；屏蔽 NFT
🦐 搞定。已收藏 2 张，已屏蔽方向：NFT。

> 最近发了什么
🦐 探索档案：5 张 / 平均 8.12 分 / 方向 4 类
    01. #005（深潜）城市生成智能      8.5
    02. #001      AI 工具使用安全对齐   8.7
    ...
```

所有交互都是文本，用户无需打开新窗口或学新操作。明信片以完整正文贴到聊天窗口，链接可直接点击。

### 2.3 关键流程

五条核心流程覆盖 90% 的用户触点：

1. **安装与首次激活** — `clawhub install wanderclaw` → 用户说"虾游" → skill 检查 `state.json` 不存在 → 问兴趣 → 跑 setup.sh + schedule-cron.sh → 4 次冷启动探索 → 送达 4 张明信片。
2. **日常定时探索** — cron 在 9/12/15/20 点自动触发，深度/轻扫交替。每次探索走六步流程：选题 → 搜索 → 阅读 → 写作 → 评分 → 归档。评分 ≥ 7 推送到聊天窗口，5–7 归档不推送。
3. **用户主动探索** — 用户说"探索"/"找点有趣的"立即出发一次；说"找点深的"进入深潜（只用深度源、字数 450–600、评分门槛 ≥ 8）。
4. **管理与反馈** — 收藏 / 屏蔽 / 回顾 / 随机翻牌 / 分享等模式，都通过关键词在聊天窗口完成，状态实时写回 `state.json`。👍/👎 反馈即时调整兴趣权重。
5. **周度汇总** — 每周日 10:00 自动生成周报：精选、方向分布、反馈统计、下周建议。

### 2.4 最终效果

随附的 [`POSTCARD-SAMPLES-20.pdf`](POSTCARD-SAMPLES-20.pdf) 收录 20 张代表性明信片 + 20 张对应人物卡，覆盖 AI Agent、生物工程、认知科学、分布式系统、城市规划、Serendipity（意外发现）六大方向，含 1 张深潜样本。评审可直接翻阅感受产出质感。

几个可观察的效果指标：

- **字数合规**：样本册 20 张明信片的中文字数均落在 300–450 区间（深潜 450–600），无一越界。
- **评分分布**：样本评分集中在 7.5–9.0，符合推送门槛策略。
- **跨信引用网络**：样本里 13 张明信片引用了其他明信片（#001、#003 等为多次被引用的"枢纽"）。半年后的用户档案会自然形成一张主题网络。
- **人物卡密度**：每张明信片都有一位具名研究者，给"谁说的"提供锚点，避免"有人说" / "据报道"式模糊叙述。

### 2.5 当前局限

Skill 形态下无法彻底解决的三个问题：

1. **LLM 稳定性依赖主 agent 侧**。在繁忙时段 idle timeout 和 gateway cron add 超时频发。
2. **可发现性**。DORMANT 模型下，用户必须自己知道说"虾游"才能触发。当前方案依赖主 agent 主动读 memory，不够可靠。
3. **软约束**。字数、格式、反幻觉只能通过 prompt 引导，无法强制。偶有边缘超限。

这些不是 SKILL.md 能单独解决的——是"把产品塞进 skill 壳子"的天花板。

## 3. 未来规划

### 3.1 定位：Skill 版持续开源，产品版闭源

Skill 版本会持续开源维护，作为信息管理策略和交互范式的参考实现。它证明了产品方向，也清晰暴露了 Skill 壳子的天花板。

闭源的产品版本并非替代 Skill，而是把它作为开源骨架，承担 Skill 无法胜任的三件事：

- **部署难度**：不再依赖用户自行安装 OpenClaw、配置模型 provider、管理 cron。开箱即用。
- **长期存储**：明信片、人物卡、兴趣图谱的长期累积与跨设备同步，由服务端保障。
- **Token 成本**：通过服务端调度和内部流程切分，在更低的综合成本下提供更稳定的产出质量。

### 3.2 产品形态

**核心逻辑云服务化** — 把信息打捞、明信片生成、主题聚合从本地 agent 迁到服务端：

- 打捞层：RSS / arXiv API / HN Firebase / Reddit / 精选播客 transcript / Newsletter 的统一聚合管道。
- 处理层：选题 / 搜索 / 阅读 / 评分 / 写作 / 人物卡 / 跨明信片关联，按阶段流水线。
- 核验层：链接可达性、原文片段抽取、作者机构交叉核对。
- 调度层：按用户兴趣图谱与权重触发，记录去重和来源多样性。

**用户端点多渠道** — 让明信片到达用户已有的信息收件口，而不是强制打开一个新 App：

- 移动端 / Web 端：阅读、档案、统计、反馈。
- 既有通道接入：飞书、邮件、RSS 输出。
- 开放 API：允许开发者把虾游的输出接入自有工作流。

### 3.3 定价与成本模型

两条可量化的成本优势，支撑商业化定价：

1. **规模效应共享打捞**
   同一主题的信息源（一篇新论文、一条 HN 热帖、一条深度 newsletter）只需一次打捞、阅读、评分，即可推送给多位感兴趣的用户。用户规模越大，单张明信片的平均成本越低。Skill 版本每次探索都要独立调 LLM，没有这个红利。

2. **内部环节模型分层**
   明信片流程可切成多个阶段，每个阶段对模型能力的要求不同。选题、搜索词生成等轻量任务用便宜模型，深度阅读和写作用 top 模型，人物卡和关联用中等模型。混合调度下，综合 token 成本相比"全程 opus"有显著下降，质量仍由关键环节保证。

定价草案：

| 档位 | 模型策略 | 月配额 | 月费 |
|---|---|---|---|
| 尝鲜 | 全流程 sonnet 级 | 每天 2–3 张 | 限量免费 |
| 日常 | sonnet + 关键环节 opus | 每天 5–8 张 + 周报 + 深潜 | ¥29–49 |
| 深度 | 全流程 top 模型 + 大 context | 每天 10 张以上 + 双倍深潜 + 跨年主题聚合 | ¥99–149 |
| 订制 | 按需 | 行业垂直管道（法律 / 医疗 / 投资等） | 企业议价 |

### 3.4 用户粘性：数据沉淀本身即资产

虾游产生的几类数据会随时间沉淀为个人化资产，构成切换成本与二次收入：

- **明信片档案**。一年后用户会拥有数百张独立、附自己反馈与注解的知识单元，类似跑步记录或日记，有情感留存。
- **兴趣图谱**。随使用逐步个人化，迁移到其他工具需要重训练。
- **知识关联网络**。跨明信片引用与主题聚合形成用户专属的认知地图，长期价值高于任何单张明信片。

这些存量本身具备商业化空间——档案导出、跨设备同步、年度精选装订、主题合集等付费增值服务，在订阅之外构成次要收入线。
