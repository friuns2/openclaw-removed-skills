# AI智能结算助手Pro - v1.1.0 更新说明

## 🎉 新增功能

### 1. 话题词精准匹配识别

支持识别视频标题中是否包含指定话题词，采用**精准匹配**策略：

#### 匹配规则
- ✅ **精准匹配**：`#金铲铲` 只能匹配 `#金铲铲`
- ❌ **部分匹配无效**：`#金铲铲` 不能匹配 `#金铲铲之战`
- ✅ **多话题共存**：标题包含 `#金铲铲` 和 `#金铲铲之战` 时，可匹配 `#金铲铲`

#### 逻辑关系支持

**且关系 (AND)**：要求同时携带所有指定话题词
```
示例：同时携带 #金铲铲 和 #攻略
触发关键词：且、和、同时、都、AND
```

**或关系 (OR)**：要求至少携带一个话题词
```
示例：携带 #金铲铲 或 #云顶之弈
触发关键词：或、OR（或无明确关键词）
```

#### 使用示例

```python
# 示例1：单个话题词
"总奖金2万元，播放量≥3万，必须携带话题 #金铲铲 的作者等额瓜分"

# 示例2：多个话题词（且关系）
"奖池5万元，播放量≥10万，同时携带话题 #金铲铲 和 #攻略 的作者瓜分"

# 示例3：多个话题词（或关系）
"总奖金3万，作品≥3条，携带话题 #英雄联盟 或 #云顶之弈 的用户瓜分"
```

### 2. 规则确认流程

在执行结算前，AI会返回格式化的规则理解供用户确认：

#### 流程说明

1. **AI解析规则** → 转换为结构化配置
2. **返回规则理解** → 格式化展示给用户
3. **用户确认/修正** → 可多轮调整
4. **确认无误后** → 开始执行结算

#### 规则理解示例

```
═══════════════════════════════════════════
📋 规则理解确认
═══════════════════════════════════════════

【奖池 1】达标瓜分奖池

  📊 结算模式: guaranteed
  ✅ 达标条件: 播放量 >= 30,000
  🏷️ 话题词要求: #金铲铲
     (逻辑关系: OR - 至少一个话题词出现)

═══════════════════════════════════════════
❓ 请确认以上规则理解是否正确？
   ✅ 回复「确认」或「正确」开始结算
   ✏️ 提出修改意见，我会更新规则理解
═══════════════════════════════════════════
```

### 核心方法

#### 话题词解析
```python
@staticmethod
def parse_topic_rule(rule_text: str) -> Optional[TopicRule]:
    """解析话题词规则"""
    # 提取所有话题词（以#开头）
    topics = re.findall(r'#[^#\s,，和或]+', rule_text)
    
    # 判断逻辑关系
    logic = "OR"  # 默认为或
    if any(keyword in rule_text for keyword in ['且', '和', '同时', 'AND', '都']):
        logic = "AND"
    
    return TopicRule(topics=topics, logic=logic)
```

#### 话题词检查
```python
def _check_topic_rule(self, titles: List[str], topic_rule: TopicRule) -> bool:
    """检查视频标题是否符合话题词规则"""
    # 检查是否有任意一个视频标题满足话题词要求
    for title in titles:
        if topic_rule.check(title):
            return True
    return False
```

## ⚠️ 注意事项

### 话题词匹配

1. **大小写敏感**：`#金铲铲` 与 `#金铲铲` 完全一致才匹配
2. **特殊字符**：话题词中不应包含空格、逗号等分隔符
3. **多视频策略**：作者只要有任意一个视频满足即可

### 规则解析

1. **关键词识别**：确保使用明确的逻辑关键词（且/或）
2. **话题词格式**：必须以 `#` 开头
3. **规则确认**：务必让用户确认后再执行结算

### 数据要求

1. **标题字段**：CSV必须包含"视频标题"或"标题"字段
2. **编码格式**：建议使用 UTF-8 编码
3. **字段命名**：支持多种变体，但建议使用标准命名
