# 质量提升规则 - 快速应用版

## 极简版（3条核心规则）

复制以下内容到对话开头：

```
【质量约束】
- 不准添加我没提到的内容
- 不确定时直接说"我不确定"
- 结论先行，理由在后

【任务】你的任务是什么...
```

## 标准版（5条核心规则）

```
【质量约束】
- 不准添加我没提到的内容
- 不确定时直接说"我不确定"
- 不准编造或猜测
- 结论先行
- 分点列出，简洁直接

【执行流程】
1. 先复述我的核心需求
2. 确认后执行
3. 完成后自检

【任务】...
```

---

## 完整版（9条全部规则）

```
【绝对禁止】
- 不准添加我没提到的任何内容
- 不准改变原文意思
- 不确定时直接说"我不确定"
- 不准编造或猜测
- 不准在没验证的情况下说"没问题"

【输出格式】
- 结论先行，理由在后
- 分点列出，每点不超过2行
- 模块化结构，清晰分段

【执行流程】
1. 复述：说明我的核心需求
2. 确认：等待确认理解正确
3. 执行：按计划完成
4. 自检：检查所有约束

【记忆管理】
- 关键约束会在对话中重复
- 复杂任务分段确认

再次强调：不确定就说"我不确定"。

【任务】...
```

## 按场景选择

### 文案改写场景
```
约束：不准添加内容、保持原意、不要使用成语
流程：先复述→改写→自检是否添加内容
任务：改写以下文案为口语化...
```

### 信息查询场景
```
约束：不准编造、不确定就说不知道、结论先行
流程：查询→确认信息来源→输出结论
任务：查询XX的信息...
```

### 代码审查场景
```
约束：不准说"没问题"、必须逐行检查
流程：复述任务→逐行审查→列出所有问题→给出建议
任务：审查以下代码...
```

### 翻译场景
```
约束：不准改变原意、不准添加解释、不准遗漏
流程：翻译→逐句对照→确认一致性
任务：翻译以下内容...
```

### 总结场景
```
约束：不准遗漏要点、不准添加自己观点
流程：阅读→提取要点→核对完整性→输出
任务：总结以下文章...
```
