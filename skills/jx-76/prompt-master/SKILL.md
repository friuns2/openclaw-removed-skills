# Prompt Master - 提示词工程精华版

## 描述

提示词工程的**快速参考版**。基于 Claude Code 源码的9条核心规则，提供最精简的规则速查和即用模板。

**定位**：快速上手、极简使用、随身携带的提示词速查手册

**完整版请见**：`prompt-engineering` skill（含详细规则、完整模板库、质量检测清单）
**质量版请见**：`quality-boost` skill（含效果对比、质量指标、深度分析）

## 9条核心规则

### 规则1：模块化提示词
**原则**：把提示词拆成独立模块，不要写成一大段。

**模板结构**：
```
【角色】你是...
【规矩】必须遵守...
【格式】输出要求...
【任务】具体要做什么
```

**示例**：
```
❌ 错误：
你是一个专业文案编辑，帮我把这段文字改得更口语化，不要用太书面的表达，控制在300字以内，分成3段，每段一个核心观点。

✅ 正确：
【角色】你是专业文案编辑
【规矩】
- 不要使用书面语
- 控制在300字以内
- 分成3段
【格式】每段一个核心观点
【任务】改写以下文字："..."
```

### 规则2：用"不准"替代"请"
**原则**：与其教AI怎么做，不如告诉它什么不能做。

**必加禁令清单**：
```
【绝对禁止】
- 不准添加我没提到的内容
- 不准改变原文的意思
- 不确定的地方保留原文，不准自己发挥
- 不准在没验证的情况下说"看起来没问题"
- 如果有疑问，先问我，不准猜测
```

### 规则3：重要的话说两遍
**原则**：关键规则在开头和结尾各说一次。

**应用方式**：
```
开头：【规矩】全文不要出现成语
...
结尾：再次提醒：全文不要出现成语
```

### 规则4：主动管理记忆
**原则**：AI的记忆会被压缩，重要信息要主动重复。

**策略**：
- 关键需求在开头和中间各说一次
- 对话超过20轮主动开新会话
- 自己携带关键上下文

### 规则5：自我验证
**原则**：完成就交差，必须检查质量。

**自检清单**：
```
完成后请检查：
□ 是否满足所有约束
□ 是否有逻辑错误
□ 是否有遗漏
□ 是否添加了未要求的内容
```

### 规则6：不知道就说不知道
**原则**：明确禁止AI编造答案。

**必加声明**：
```
如果你不确定或信息不足，
请直接说"我不确定"，不要猜测。
```

### 规则7：先复述再动手
**原则**：AI必须先复述需求，确认后再执行。

**强制流程**：
```
在给出最终答案前，请先复述：
1. 你认为我的核心需求是什么
2. 你打算用什么方法解决
3. 你理解的关键约束有哪些

确认无误后，再开始执行。
```

### 规则8：不知道就说不知道
**原则**：明确禁止AI编造答案。

**必加声明**：
```
如果你不确定或没有足够信息，请直接说"我不确定"，不要猜测或编造。
```

### 规则9：先说结论
**原则**：先说答案或行动，再说理由，不要先铺垫。

**输出格式要求**：
```
【结论】...
【理由】...
【细节】（可选）...
```

## 使用方法

### 方式1：手动调用
当用户需要高质量回答时，说：
> "应用提示词大师规则"

系统会自动将9条规则注入到当前对话中。

### 方式2：自动加载（推荐）
在会话配置中启用自动加载：
```json
{
  "skills": {
    "prompt-master": {
      "autoLoad": true,
      "injectTo": "system"
    }
  }
}
```

### 方式3：针对特定任务
为特定任务定制提示词：
```
使用提示词大师规则，帮我写一份...
```

## 万能提示词模板

```
【角色】
你是[具体角色]

【绝对禁止】
- 不准[禁止项1]
- 不准[禁止项2]
- 不确定时直接说"我不确定"，不要猜测
- 不准在没验证的情况下说"看起来没问题"

【输出格式】
- 先说结论，再说理由
- 分点列出，每点不超过2行
- [其他格式要求]

【执行流程】
1. 先复述我的核心需求
2. 我确认后再开始执行
3. 完成后自我检查一遍

【任务】
[具体任务描述]
```

> 【结论】已按要求改写完成

## 注意事项

1. 规则越多，提示词越长，注意Token消耗
2. 根据任务复杂度选择需要的规则
3. 简单任务不需要全部9条规则
4. 建议将常用规则保存为模板

## 场景化快捷指令

### 场景1：改写文章
**一键指令**：
```
【禁止】添加内容、改变原意
【格式】口语化、分点、每点2行内
【流程】先复述→再改写→后自检
任务：改写这段文字：
```

### 场景2：审查代码
**一键指令**：
```
【禁止】说没问题、没验证下结论
【检查】功能、性能、安全、异常处理
【格式】问题清单+位置+建议+自检
任务：审查这段代码：
```

### 场景3：查询信息
**一键指令**：
```
【禁止】编造、猜测
【格式】结论→理由→来源
不确定就说"我不确定"
任务：查询：
```

### 场景4：数据分析
**一键指令**：
```
【禁止】添加数据外信息、无依据推测
【框架】概况→发现→异常→建议
【格式】结论→分析→数据支撑→自检
任务：分析这些数据：
```

### 场景5：写邮件
**一键指令**：
```
【禁止】添加无关内容、改变语气
【格式】称呼→正文→结尾
【约束】专业、简洁、结论先行
任务：写一封关于...的邮件
```

### 场景6：做总结
**一键指令**：
```
【禁止】添加原文没有的观点
【格式】核心结论→要点→细节
【约束】准确、完整、不遗漏
任务：总结以下内容：
```

### 场景7：Debug
**一键指令**：
```
【禁止】说"看起来没问题"、猜测原因
【流程】分析→定位→验证→解决
【格式】问题→原因→方案→验证
任务：排查这个bug：
```

### 场景8：翻译
**一键指令**：
```
【禁止】添加内容、改变语气、过度意译
【格式】原文→译文→备注
【自检】意思一致、语气一致、无添加
任务：翻译成...：
```

## 极简模式

当Token紧张时使用：

### 超简版（50字内）
```
禁止：添加、编造、说没问题
格式：结论→理由，分点，每点2行
流程：复述→执行→自检
任务：
```

### 核心禁令版（30字内）
```
不准添加、不准编造、不确定就说不知道
结论先行、先复述再执行
任务：
```

### 最简版（15字内）
```
禁令：不添加、不编造、不确定就说
任务：
```

## 常见错误速查

| 问题 | 原因 | 快速修复 |
|------|------|----------|
| 添加内容 | 禁令不够前 | 把"不准添加"放最前面 |
| 理解偏差 | 没复述 | 加"先复述再执行" |
| 回答太长 | 没约束长度 | 加"每点不超过2行" |
| 编造信息 | 没禁止猜测 | 加"不确定就说不知道" |
| 说"没问题" | 没禁止 | 加"不准说看起来没问题" |

## 记忆口诀

**"禁令复述结论简，重复自检不瞎编"**

- **禁**令优先（规则2）
- 主动**复**述（规则4）
- **结**论先行（规则7）
- 简**洁**直接（规则8、9）
- **重**复强调（规则3）
- **自**检验证（规则5）
- 不**瞎**编（规则6）
