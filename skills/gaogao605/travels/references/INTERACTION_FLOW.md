# 分贝通交互流程参考

## 核心原则

1. **后台静默认证** - 认证检查在后台执行，不在前台向用户展示认证状态
2. **需求优先** - 先了解用户需求，再执行搜索
3. **结果导向** - 直接展示结果，让用户选择

---

## 标准流程

### 需求完整 → 直接搜索

```
用户输入: 北京三元桥附近酒店 明天入住 预算500

✅ 正确做法:
   🔍 正在查询三元桥附近、500元以内的酒店...
   [展示酒店列表]

❌ 错误做法:
   ✅ 酒店已认证（剩余64天），直接查询：
   🔍 正在查询...
```

### 需求不完整 → 先询问

```
用户输入: 北京酒店

AI响应:
   请补充以下信息：
   - 入住区域？（如：三元桥、国贸）
   - 入住日期？（明天/具体日期）
   - 预算范围？（可选）
```

---

## 禁止事项

| 禁止行为 | 原因 |
|---------|------|
| 展示认证状态 | 用户关心的是结果，不是技术细节 |
| 先搜索后询问 | 应先确认需求再搜索，避免无效查询 |
| 输出脚本日志 | 如"正在向 138xxxx 发送验证码" |
| 显示后台命令 | 如"python3 scripts/auth.py" |

---

## 推荐话术

### 搜索开始
```
🔍 正在查询[区域][预算条件]的酒店...
```

### 需求确认
```
请补充以下信息：
- [缺失项1]
- [缺失项2]
```

### 认证引导（仅未认证时）
```
⚠️ 需要先登录分贝通账号
请提供手机号完成认证
```

### 结果展示
```
[酒店列表]

---
💡 回复 **序号** 查看房型详情
```
