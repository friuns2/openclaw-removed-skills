# 道法术器认知框架

「道法术器」是中国传统哲学中的认知模型，用于系统性地理解和解决问题。这套框架可以帮助我们从一个想法出发，构建完整的执行方案。

---

## 一、四层认知结构

### 道（底层规律）

**定义**：事物的本质规律和底层逻辑。

**核心问题**：这件事的本质是什么？背后的规律是什么？

**作用**：
- 把握正确方向
- 避免战略错误
- 理解事物本质

**示例**：
| 领域 | 道（底层规律） |
|------|---------------|
| 直播带货 | 注意力经济，信任转化 |
| 学习技能 | 刻意练习，反馈迭代 |
| 减肥 | 能量守恒，可持续性 |
| 投资理财 | 风险收益对等，复利效应 |

---

### 法（方法体系）

**定义**：可复用的方法论和系统框架。

**核心问题**：用什么方法/系统来做这件事？

**作用**：
- 提供可操作的框架
- 避免重复造轮子
- 保证执行的系统性

**示例**：
| 领域 | 法（方法体系） |
|------|---------------|
| 直播带货 | 人货场模型、选品方法论、话术框架 |
| 学习技能 | 费曼学习法、PDCA循环、OKR目标法 |
| 减肥 | 间歇性断食法、低碳饮食法、运动燃脂法 |
| 投资理财 | 定投策略、资产配置、价值投资 |

---

### 术（执行技法）

**定义**：具体的执行步骤和技巧。

**核心问题**：第一步做什么？第二步？具体怎么操作？

**作用**：
- 提供可执行的行动计划
- 明确时间节点和里程碑
- 量化任务和完成标准

**示例**：
| 领域 | 术（执行技法） |
|------|---------------|
| 直播带货 | 第1周：账号搭建 → 第2周：内容测试 → 第3周：直播试水 |
| 学习技能 | 第1月：基础入门 → 第2月：专项练习 → 第3月：实战应用 |
| 减肥 | 第1阶段：饮食调整 → 第2阶段：运动启动 → 第3阶段：习惯固化 |
| 投资理财 | 第1步：风险评估 → 第2步：品种选择 → 第3步：定期执行 |

---

### 器（工具推荐）

**定义**：承载技法的具体工具和平台。

**核心问题**：用什么工具最合适？

**作用**：
- 提升执行效率
- 降低技术门槛
- 标准化操作流程

**示例**：
| 领域 | 器（工具推荐） |
|------|---------------|
| 直播带货 | OBS推流、抖音创作者中心、蝉妈妈数据 |
| 学习技能 | Anki记忆卡、Notion笔记、在线课程平台 |
| 减肥 | MyFitnessPal记录、Apple Watch监测、健身房器材 |
| 投资理财 | 蚂蚁财富、雪球、券商APP |

---

## 二、天时地利人和

在深度规划中，我们引入「天时地利人和」三个维度，与道法术器结合使用。

### 天时（时机趋势）

**核心问题**：现在做是不是好时机？

**分析维度**：
- 行业发展趋势
- 政策环境变化
- 市场窗口期
- 技术成熟度

**评分标准**（1-10分）：
- 9-10分：黄金窗口期，机会难得
- 7-8分：好时机，可以入场
- 5-6分：时机一般，需要差异化
- 3-4分：时机不佳，风险较高
- 1-2分：不建议入场

---

### 地利（环境资源）

**核心问题**：在什么环境/平台做最合适？

**分析维度**：
- 可用资源盘点
- 平台/渠道选择
- 地域/场景优势
- 资源缺口分析

**评分标准**（1-10分）：
- 9-10分：资源充足，优势明显
- 7-8分：资源较好，可以起步
- 5-6分：资源一般，需要补短板
- 3-4分：资源不足，需要筹备
- 1-2分：严重缺乏，建议暂缓

---

### 人和（道法术器）

**核心问题**：个人的能力、方法、执行是否到位？

这就是「道法术器」四层框架的应用：
- **道**：认知是否到位
- **法**：方法是否正确
- **术**：执行是否到位
- **器**：工具是否趁手

---

## 三、锚点层级

当用户提出一个想法时，我们会判断这个想法落在哪个层级，然后补齐其他层级。

### 四种锚点类型

| 锚点 | 特征 | 补齐策略 |
|------|------|---------|
| **道** | 有明确的规律认知，但缺乏方法 | 补齐法、术、器 |
| **法** | 有方法论框架，但缺乏具体执行 | 补齐道（验证）、术、器 |
| **术** | 有具体行动计划，但缺乏方法论 | 补齐道、法（提炼）、器 |
| **器** | 有工具/平台，但缺乏整体规划 | 补齐道、法、术 |

### 判断示例

| 用户输入 | 锚点判断 | 理由 |
|---------|---------|------|
| "直播的本质是信任经济" | 道 | 认知层面，有规律总结 |
| "我想用费曼学习法" | 法 | 方法层面，有方法论 |
| "我计划第一周做账号搭建" | 术 | 执行层面，有具体步骤 |
| "我想用OBS做直播" | 器 | 工具层面，有具体工具 |
| "我想做直播" | 无锚点 | 目标层面，需要从头规划 |

---

## 四、迭代诊断

在执行过程中，如果遇到问题，我们可以诊断问题出在哪个层级。

### 诊断维度

| 问题层级 | 典型问题 | 诊断标志 |
|---------|---------|---------|
| **天时** | 趋势判断错误、时机不对 | 行业整体下行、政策限制 |
| **地利** | 平台选择错误、资源不足 | 流量获取困难、资源耗尽 |
| **道** | 方向错误、认知偏差 | 越努力越偏离目标 |
| **法** | 方法不当、框架失效 | 方法执行了但没效果 |
| **术** | 执行不到位、节奏不对 | 知道怎么做但没做好 |
| **器** | 工具不当、效率低下 | 工具限制、操作困难 |

### 调整策略

1. **天时问题**：调整时机预期，等待或加速
2. **地利问题**：换平台/渠道，补充资源
3. **道的问题**：重新理解本质，调整方向
4. **法的问题**：更换方法论，优化框架
5. **术的问题**：细化执行，调整节奏
6. **器的问题**：更换工具，提升技能

---

## 五、框架应用流程

```
用户想法
    ↓
判断锚点层级
    ↓
分析天时地利
    ↓
补齐道法术器
    ↓
输出完整方案
    ↓
执行 → 遇到问题 → 诊断层级 → 调整优化
```

---

## 六、框架价值

1. **系统性**：不遗漏任何层级
2. **层次性**：从本质到工具层层递进
3. **可操作性**：每个层级都有具体输出
4. **可迭代**：问题可以追溯到具体层级
5. **通用性**：适用于任何领域的规划

---

*此框架源自中国传统哲学智慧，结合现代管理理念，形成了一套完整的认知和执行体系。*
