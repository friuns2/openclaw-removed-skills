## 案例一：我想做直播带货

### 命令

```bash
node scripts/moudao.js --quick "我想做直播带货"
```

### 输出

```markdown
## 天时（时机趋势）

**评分**: 7/10

- **趋势**: 直播电商进入成熟期，垂直细分领域仍有增量空间
- **时机**: 现在入场不算晚，但红利期已过，需要差异化定位
- **窗口期**: 未来6-12个月是最佳入场窗口
- **风险**: 平台政策收紧、流量成本上升、同质化竞争

## 地利（环境资源）

**评分**: 6/10

- **资源分析**: 需要评估现有资源（货源、场地、设备、时间）
- **推荐平台**: 抖音（流量大）/ 视频号（私域强）/ 小红书（种草转化）
- **优势**: 进入门槛低、平台扶持新手、变现路径清晰
- **短板**: 需要持续投入、竞争激烈、需要学习成本

> **锚点层级**: 目标 — 用户有明确目标，需要完整规划

## 道（底层规律）

- **信任经济**: 直播的本质是建立信任，转化源于信任
- **内容为王**: 有价值的内容才能留住用户
- **人货场匹配**: 主播人设、货品选择、场景搭建需要统一

**常见误区**:
- ❌ 颜值高就能火 → ✅ 内容价值才是核心
- ❌ 低价才能卖出去 → ✅ 信任度决定转化率
- ❌ 每天播就能赚钱 → ✅ 策略性运营才能起号

## 法（方法体系）

### 垂直定位法
> 适合: 新手起步

选择一个细分领域深耕，建立专业人设

**步骤**: 选领域 → 找对标 → 定风格 → 建人设

### 起号冷启动法
> 适合: 零基础起步

通过短视频引流，逐步过渡到直播

**步骤**: 短视频测试 → 积累粉丝 → 直播试水 → 放大投入

### 话术转化框架
> 适合: 提升转化率

痛点唤醒 → 产品介绍 → 信任背书 → 价格锚点 → 限时促单

**步骤**: 设计话术 → 练习内化 → 直播测试 → 优化迭代

## 术（执行技法）

### 起号期（第1-4周）

| 任务 | 耗时 | 完成标准 |
|------|------|----------|
| 账号搭建 | 2h | 完成主页装修、简介、头像 |
| 领域调研 | 4h | 分析10个对标账号 |
| 设备准备 | 3h | 灯光、支架、收音设备到位 |
| 短视频试水 | 8h/周 | 发布7条测试内容 |

### 试播期（第5-8周）

| 任务 | 耗时 | 完成标准 |
|------|------|----------|
| 直播测试 | 2h/天 | 完成10场直播 |
| 数据复盘 | 1h/场 | 记录并分析每场数据 |
| 选品测试 | 3h/周 | 测试3-5个品类 |
| 话术优化 | 2h/周 | 迭代销售话术 |

### 放量期（第9-12周）

| 任务 | 耗时 | 完成标准 |
|------|------|----------|
| 增加场次 | 4h/天 | 稳定日播节奏 |
| 供应链对接 | 4h/周 | 建立3个稳定货源 |
| 投流测试 | 2h/周 | 测试付费流量ROI |
| 团队搭建 | 4h/周 | 确定运营/客服分工 |

## 器（工具推荐）

### 直播推流

- **OBS Studio**（免费）: 专业推流软件，支持多场景切换
- **抖音直播伴侣**（免费）: 平台官方工具，操作简单

### 数据分析

- **蝉妈妈**（付费）: 达人数据分析，竞品监控
- **飞瓜数据**（付费）: 直播数据追踪，商品分析
- **抖音创作者中心**（免费）: 基础数据查看

### 商品管理

- **精选联盟**（免费）: 平台选品中心
- **淘宝联盟**（免费）: 淘系商品推广

### 辅助工具

- **剪映**（免费）: 短视频剪辑
- **美颜相机**（免费）: 直播美颜
- **提词器APP**（免费）: 话术提示
```

## 案例二：我想学编程

### 命令

```bash
node scripts/moudao.js --deep "我想学编程" --profile "上班族，每周10小时，零基础"
```

### 输出

```markdown
## 天时（时机趋势）

**评分**: 8/10

- **趋势**: AI时代编程门槛降低，但核心逻辑思维价值上升
- **时机**: 现在是学习编程的好时机，资源丰富，工具成熟
- **窗口期**: AI辅助编程窗口期，抓住红利
- **风险**: 技术更新快，需要持续学习

## 地利（环境资源）

**资源分析**: 上班族，每周10小时，时间有限但稳定，零基础需要系统学习

- **推荐平台**: 
  - 入门：B站免费教程
  - 进阶：极客时间、Coursera
  - 实践：GitHub、LeetCode

- **优势**: 目标明确、时间稳定、有职业背景
- **短板**: 时间有限、零基础、缺乏环境

> **锚点层级**: 目标 — 需要完整规划学习路径

## 道（底层规律）

- **刻意练习**: 编程能力 = 练习量 × 反馈质量
- **项目驱动**: 在做中学，用真实项目驱动学习

**常见误区**:

- ❌ 只看不练 → ✅ 动手写代码是唯一出路

## 法（方法体系）

以完成具体项目为目标，倒推需要学习的知识点

**步骤**: 选项目 → 拆解知识点 → 逐一学习 → 整合完成

**步骤**: 识别弱项 → 设计练习 → 即时反馈 → 修正迭代

用自己的话复述学到的知识，发现理解盲区

**步骤**: 学习概念 → 尝试复述 → 发现盲区 → 回归学习
```

## 案例三：迭代诊断

### 命令

```bash
node scripts/moudao.js --diagnose "直播做了两周，每天播3小时，但场观只有100人左右，几乎没有人下单" --context "我想做直播带货"
```

### 输出

```markdown
## 诊断结论

问题出在**术的层面**：执行策略需要优化，特别是流量获取和转化环节。

## 问题分析

| 层级 | 问题 | 建议 |
|------|------|------|
| 术 | 直播时段可能不对 | 测试不同时段，找到流量高峰 |
| 术 | 没有短视频引流 | 直播前发布预热短视频 |
| 术 | 话术可能不够吸引 | 优化开场话术，前5分钟留人 |
| 器 | 数据分析不足 | 使用蝉妈妈分析竞品数据 |
| 法 | 冷启动方法错误 | 应该先积累粉丝再开播 |

## 调整建议

- **调整前**: 直接开播，每天3小时，无短视频引流
- **调整后**: 
  1. 每天发1-2条短视频积累粉丝
  2. 直播前发布预热视频
  3. 调整直播时段到晚间8-10点
  4. 优化开场话术，5秒内抛出利益点
  5. 每周复盘数据，持续优化
```

## 案例四：我想减肥

### 命令

```bash
node scripts/moudao.js --quick "我想减肥，目标减10斤"
```

### 输出

```markdown
## 道（底层规律）

- **能量守恒**: 减肥本质是制造热量缺口
- **可持续性**: 能坚持的方法才是好方法
- **代谢适应**: 身体会适应低热量状态

**常见误区**:
- ❌ 节食就能瘦 → ✅ 合理饮食+运动才是正道
- ❌ 减得越快越好 → ✅ 每周0.5-1kg是健康速度
- ❌ 只看体重 → ✅ 体脂率和围度更重要

## 法（方法体系）

### 热量缺口法
> 适合: 大多数人

制造每日300-500卡热量缺口

**步骤**: 计算TDEE → 设定目标热量 → 饮食调整 → 运动配合

### 间歇性断食法
> 适合: 作息规律的人

16:8断食法，在8小时内吃完一天的食物

**步骤**: 选择进食窗口 → 适应期 → 长期坚持

## 术（执行技法）

### 第一阶段（第1-4周）

| 任务 | 耗时 | 完成标准 |
|------|------|----------|
| 饮食记录 | 10min/天 | 连续记录28天 |
| 运动启动 | 30min/天 | 每周运动4次 |
| 睡眠调整 | - | 保证7小时睡眠 |

### 第二阶段（第5-12周）

| 任务 | 耗时 | 完成标准 |
|------|------|----------|
| 饮食优化 | - | 形成健康饮食习惯 |
| 运动进阶 | 45min/天 | 加入力量训练 |
| 数据监测 | - | 每周记录体重和围度 |

## 器（工具推荐）

### 饮食记录

- **MyFitnessPal**（免费）: 热量计算，食物库丰富
- **薄荷健康**（免费）: 中文友好，本地食物库

### 运动辅助

- **Keep**（免费/付费）: 健身课程，跟练方便
- **Apple Watch/小米手环**（付费）: 运动监测

### 数据追踪

- **体重秤**（付费）: 每日体重监测
- **体脂秤**（付费）: 体脂率追踪
```
