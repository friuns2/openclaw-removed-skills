## 常见问题

### Q1: context-booster 是什么？

**A**: context-booster 是一个上下文增强引擎，能够：
- 捕获执行上下文（技能执行前后的完整上下文）
- 智能压缩历史对话，保留关键信息
- 基于语义检索相关上下文
- 在执行前注入相关上下文
- SQLite 存储，支持跨会话持久化

核心价值是解决 AI 上下文窗口限制，保持长对话的连贯性。

### Q2: 什么情况下应该使用 context-booster？

**A**: 以下场景适合使用 context-booster：
- ✅ 长对话需要保持连贯性（>10 轮）
- ✅ 复杂任务需要多步骤上下文
- ✅ 跨会话继续未完成任务
- ✅ 上下文窗口即将耗尽（>70%）

**不适合的场景**：
- ❌ 短对话（<10 轮）
- ❌ 简单任务（单步骤）
- ❌ 敏感信息（需特殊处理）

### Q3: 记忆压缩的原理是什么？

**A**: context-booster 使用语义压缩：
1. **提取关键信息** - 识别对话中的决策、行动项、重要事实
2. **语义向量化** - 将信息转换为向量表示
3. **相似度聚类** - 将相似信息合并
4. **生成摘要** - 用更少的 tokens 表达相同含义

典型压缩率：3:1 到 5:1

### Q4: 如何检索相关上下文？

**A**: 使用以下命令格式：
```
检索相关上下文：
[关键词或主题]
```

例如：
```
检索相关上下文：
技能进化方案
```

系统会返回最相关的 5 条上下文记录。

### Q5: 上下文存储在哪里？

**A**: 上下文存储在 SQLite 数据库中：
- 位置：`~/.openclaw/context-booster/context.db`
- 自动清理：30 天未访问的上下文会被标记为过期
- 手动清理：`context-booster cleanup --older-than 7d`

### Q6: 跨会话上下文如何工作？

**A**: context-booster 支持跨会话上下文：
1. 每次会话结束时自动保存关键上下文
2. 新会话开始时可以检索历史上下文
3. 使用 `检索上次会话的上下文` 命令恢复

### Q7: 上下文检索的准确率如何？

**A**: 基于内部测试：
- 检索准确率：≥85%
- 响应时间：<500ms
- 关键信息保留率：≥90%

准确率取决于：
- 关键词的明确性
- 上下文的丰富程度
- 语义向量的质量

### Q8: 如何查看已存储的上下文？

**A**: 使用以下命令：
```
查看已存储的上下文
```

会显示：
- 会话 ID
- 时间范围
- 关键词
- 重要性评分
- 访问次数

### Q9: 支持上下文导出吗？

**A**: 当前版本（v0.1.0）支持手动导出：
```
导出上下文：
[会话 ID 或关键词]
--format=markdown
--output=./export.md
```

支持的格式：
- Markdown
- JSON
- Plain Text

### Q10: 隐私和安全如何保障？

**A**: context-booster 的隐私保障措施：
- 本地存储：所有数据存储在本地 SQLite
- 不上传云端：不会将上下文发送到外部服务器
- 手动清理：可以随时删除敏感上下文
- 加密选项：企业版支持数据库加密

## 性能优化建议

### 提高检索准确率
- 使用明确的关键词
- 避免过于宽泛的查询
- 添加时间范围限制

### 减少存储占用
- 定期清理过期上下文
- 降低压缩保留率（默认 30%）
- 设置更短的过期时间

### 加快响应速度
- 减少检索结果数量（默认 5 条）
- 提高相关性阈值（默认 0.7）
- 使用缓存（企业版）
