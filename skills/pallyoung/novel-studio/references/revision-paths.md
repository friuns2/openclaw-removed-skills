# Revision Paths

## 1. Goal

Choose an effective repair order instead of revising randomly.

Use this reference when the agent has already identified problems and now must decide:
- what to fix first
- what can wait
- which local issues are actually caused by upstream problems

---

## 2. Core principle

Fix upstream causes before downstream symptoms.

Do not polish language before deciding whether the chapter has dramatic movement.
Do not tweak dialogue before deciding whether the character has a distinct inner engine.
Do not patch side scenes before deciding whether the subplot has any real function.

---

## 3. Default repair order

Prefer this order unless there is a strong reason not to:

1. Core story engine
2. Main-line / side-line structure
3. Protagonist and core-cast attraction
4. Relationship tension and scene pressure
5. Chapter-level dramatic movement
6. Language / rhetoric / polish

---

## 4. Repair path by failure type

### 4.1 If the story feels flat
Repair order:
1. check conflict engine
2. check line diversity (main line + relationship / suspense / pressure)
3. check payoff timing
4. only then refine prose

### 4.2 If the protagonist feels weak
Repair order:
1. define desire, wound, contradiction, and cost
2. define scene-level pressure that reveals those traits
3. differentiate speech and reactions
4. then revise descriptive texture

### 4.3 If side plots feel useless
Repair order:
1. define what consequence they should change
2. decide where they feed back into the main line
3. remove or combine dead branches
4. only then strengthen local scenes

### 4.4 If chapters feel readable but dead
Repair order:
1. identify missing dramatic movement
2. add cost / reveal / relationship shift / emotional shift
3. strengthen ending pull
4. then improve language efficiency

### 4.5 If prose feels weak but the structure works
Repair order:
1. keep structure stable
2. sharpen scene purpose
3. adjust rhetoric density and rhythm
4. refine image and emotional precision

---

## 5. Small-fix vs deep-fix rule

For each issue, identify:
- smallest effective fix
- medium fix
- deep structural fix

Use the smallest effective fix when:
- the foundation is sound
- the deadline is short
- the problem is local

Use the deep fix when:
- the same weakness repeats across chapters
- the problem is caused by role design or outline design
- local patching has already failed

---

## 6. Anti-chaos rule

Do not revise everything at once.
At any given moment, define:
- top priority fix
- secondary fix
- later polish items

If too many changes are applied simultaneously, continuity and voice often become less stable.

---

## 7. Output pattern

When proposing revision paths, report:
- primary problem
- upstream cause
- revision order
- smallest effective fix
- strongest recommended fix
- what not to waste time fixing yet
