# Review Severity

## 1. Goal

Classify review findings by severity so feedback is not presented as a flat list.

Use this reference when reviewing outlines, character packages, chapters, batches, or project-level materials.

The purpose is to answer:
- which problem is fatal
- which problem blocks advancement
- which problem weakens attraction but can wait
- which problem is only polish

---

## 2. Core principle

Do not treat all flaws as equal.

A weak metaphor and a broken protagonist engine are not the same class of problem.
A dead side plot and a missing comma are not the same class of problem.

Always classify the issue before proposing revisions.

---

## 3. Severity levels

### S-level: Fatal / gate-blocking
Meaning:
- the current material should not advance
- the core engine is broken or missing
- later repair will be far more expensive if ignored now

Examples:
- no usable main conflict engine
- protagonist has no attraction beyond labels
- side plots do not connect to consequence at all
- chapters are readable but dramatically inert across the batch
- outline cannot support drafting
- OOC collapse that destroys a core role

Recommended response:
- stop advancement
- name the issue directly
- propose structural repair first

### A-level: Major structural weakness
Meaning:
- material is usable only with caution
- it may technically continue, but the work will likely weaken fast if unfixed

Examples:
- relationship line exists but has weak pressure
- reveals are too soft to support suspense promises
- chapter hooks are consistently underpowered
- core supporting cast is too interchangeable
- mid-story escalation is thin

Recommended response:
- mark as priority fix
- repair before scaling further if possible

### B-level: Noticeable weakness
Meaning:
- not immediately fatal, but clearly lowering quality or attraction

Examples:
- some scenes over-explain emotion
- rhetoric density is wrong for scene speed
- a subplot payoff lands weakly but still exists
- one important role lacks a memorable speech rhythm

Recommended response:
- note clearly
- fix after structural issues

### C-level: Cosmetic / polish issue
Meaning:
- wording, texture, rhythm, or local refinement issue
- does not decide whether the material fundamentally works

Examples:
- line-level awkwardness
- repeated phrasing
- overlong paragraph shaping
- local image sharpening opportunities

Recommended response:
- only address after higher-level issues are stable

---

## 4. Mandatory review behavior

When reviewing, do not only list problems.
For each important issue, state:
- severity level
- what it damages
- whether it blocks advancement
- what kind of repair it needs

---

## 5. Anti-failure rule

Do not waste revision energy on C-level polish when S-level or A-level issues still dominate.

If the story engine is weak, do not pretend the real problem is just language refinement.
If the protagonist is dead on the page, do not hide behind “could be more vivid.”

Name the real problem.
