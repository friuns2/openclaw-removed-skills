# Hot Search Scan

## 1. Goal

Capture current market signals before formal topic recommendation begins.

This stage exists to prevent empty topic analysis based only on intuition or stale model knowledge.

---

## 2. Required input

You must have at least one of the following:
- a rough genre/topic area
- a rough audience direction
- explicit permission to begin with broad market scanning first

---

## 3. Forbidden to start if

Do not start if:
- the project is so undefined that even broad category scanning is impossible
- user constraints are so contradictory that no useful scan scope can be defined

---

## 4. Search priority

Prefer fresh external signals over internal memory.

Recommended order:
1. platform-visible lists and current rankings
2. search-engine results about recent trends
3. public discussion and analysis sources
4. model fallback only when fresh data is unavailable

If falling back to model knowledge, clearly mark it as non-real-time inference.

---

## 5. Required scan dimensions

The scan should cover at least these when relevant:
- current hot genres / tags
- platform differences (e.g. 起点 / 晋江 / 番茄 / 短剧方向)
- emotional-value direction
- adaptation / short-drama signal
- saturation risk
- differentiation opportunities

---

## 6. Required output

This stage must produce a scan artifact written into project files.

Recommended artifact:
- `00A_热点扫描.md`

The artifact should record:
- search scope
- main platforms checked
- hot directions found
- notable tags or trend signals
- high-level interpretation
- uncertainty note if data is weak

---

## 7. Completion standard

This stage is complete only if:
- a readable market-signal summary exists
- the scan identifies at least a few usable directions or warnings
- the output is written to a file, not only kept in chat

---

## 8. Do not advance if

Do not advance to market-research decision work if:
- no scan artifact exists
- the scan is too vague to inform recommendation
- all conclusions are still unsupported guesswork

---

## 9. Rollback condition

Return to hot-search scan if:
- the user changes platform or audience target significantly
- the current scan proves too weak to support recommendation
- the recommendation stage reveals stale or misaligned market assumptions

---

## 10. Quality bar

A valid hot-search scan is:
- fresh enough to matter
- concrete enough to guide later decision work
- written down in a reusable form

Invalid output includes:
- generic statements like “this is popular now” with no scan basis
- pure memory-based trend claims presented as current market fact
- chat-only scan results with no saved artifact
