# Style Bible

## Goal

Lock the project voice before long-form drafting begins.

Style is not a mood word. It is a production contract.

## Required baseline

Create `01A_风格圣经.md` and treat the approved version as the canonical voice contract.

At minimum it should define:
- lane and target reader
- platform mode
- POV and narration distance
- sentence density
- dialogue ratio
- rhythm density
- emotional / 爽点 cadence
- forbidden cliches
- allowed drift range

## Drift control

### V1
The first approved style bible is `V1`.

Once `V1` is approved:
- drafting follows it
- polishing protects it
- proofreading checks for drift against it

## 风格漂移

Style drift is a structural risk, not a cosmetic issue.

Common drift symptoms:
- every character starts sounding like the same neutral narrator
- later batches become generic or template-heavy
- the rhythm softens and loses lane identity
- child agents gradually replace the original voice with safe prose

## Revision path

If style must change, do not let batches drift silently.

Use a formal 风格修订单:
- explain why the current voice is no longer right
- specify what changes and what must remain
- version the style bible from `V1` to `V2`
- route the change through user approval
