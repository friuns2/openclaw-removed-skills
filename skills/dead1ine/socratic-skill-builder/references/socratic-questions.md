# 苏格拉底问题库（按技能类型分类）

## 通用起始问题（适用于所有技能）

1. **本质追问**：「这个技能要解决的**最核心**问题是什么？能不能用一句话说清楚？」
2. **边界追问**：「这个问题**不应该**由这个技能来解决的部分是什么？」
3. **价值追问**：「如果这个技能存在，它节省了你现在多少时间？」
4. **触发追问**：「用户最可能用哪句话触发这个技能？」
5. **失败追问**：「这个技能**不应该**在什么情况下启动？」

## 按技能类型的问题模板

### 📝 内容生成类技能

| 问题 | 目的 |
|------|------|
| 生成内容的**来源**是什么？（数据库/网络/用户输入） | 确定输入边界 |
| 生成内容的**格式**有要求吗？（Markdown/HTML/纯文本/文件） | 确定输出规范 |
| 是否有**风格/语气**要求？（正式/口语/技术/文学） | 质量标准 |
| 如何判断生成结果是**成功的**？ | 验收标准 |
| 是否有**禁止出现**的内容或词汇？ | 安全/合规边界 |

### 🔧 自动化执行类技能

| 问题 | 目的 |
|------|------|
| 这个流程目前是**手动**还是**半自动**的？ | 现状评估 |
| 执行的**触发条件**是什么？（定时/事件/手动） | 启动条件 |
| 执行结果需要**通知**谁吗？ | 反馈机制 |
| 每一步的**正确性**如何验证？ | 质量控制 |
| 如果中间一步失败，**之前已执行的部分**如何处理？ | 事务性 |

### 📊 数据分析/处理类技能

| 问题 | 目的 |
|------|------|
| 原始数据从哪里来？（文件/API/数据库） | 输入来源 |
| 数据的**规模**大概是多少？（行数/文件数/频率） | 性能要求 |
| 分析结果的**受众**是谁？（自己/团队/客户） | 输出深度 |
| 需要保留**中间过程**还是只输出最终结果？ | 完整 vs 简洁 |
| 是否有**异常值处理**的要求？ | 容错设计 |

### 🌐 网络/爬虫类技能

| 问题 | 目的 |
|------|------|
| 目标网站的**结构是否稳定**？（是否可能经常变更） | 维护成本 |
| 是否涉及**登录/验证码**？ | 技术可行性 |
| 请求的**频率**和**并发**要求？ | 礼貌爬取 |
| 抓取的内容**储存**在哪里？（文件/数据库/内存） | 输出路由 |
| 是否需要**增量更新**能力？ | 增量 vs 全量 |

### 📁 文件处理类技能

| 问题 | 目的 |
|------|------|
| 支持哪些**文件格式**？（PDF/Word/Excel/图片） | 输入边界 |
| 文件的**大小上限**是多少？ | 性能边界 |
| 处理后的文件**命名规则**是什么？ | 输出规范 |
| 是否需要**保留原文件**？ | 备份需求 |
| 批量处理时，**失败的文件**如何处理？ | 容错设计 |

### 💬 对话/客服类技能

| 问题 | 目的 |
|------|------|
| 核心是**回答问题**还是**执行任务**？ | 技能定位 |
| 回答的知识来源是什么？（文档/数据库/网络） | 知识边界 |
| 是否需要**记忆上下文**？（多轮对话） | 状态管理 |
| 遇到无法回答的问题时，**回复策略**是什么？ | 兜底方案 |
| 回答的**语气/风格**要求？ | 品牌一致性 |

## 追问技巧

### 降维追问
当用户回答模糊时，用更具体的问题拆解：
> 「你说'处理数据'——具体是**读取**数据、**转换**格式、还是**分析**后生成报告？」

### 升维追问
当用户陷在细节里时，拉回全局：
> 「回到最初的问题——你觉得这个技能**最大的价值**是什么？」

### 反面追问
从反面确认边界：
> 「如果用户要求这个技能做** X **，你觉得应该拒绝吗？」

### 假设追问
探索可能性：
> 「如果这个技能可以**完全自动化**，不需要你任何介入——你觉得它现在还不能做到的原因是什么？」
