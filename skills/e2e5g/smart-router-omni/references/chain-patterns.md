# Chain Patterns

Recommend a chain when one skill cannot cleanly finish the job.

## Common chains

- research -> analyze -> write
- research -> write -> publish
- diagnose -> implement -> verify
- capture -> organize -> retrieve
- scope -> build -> automate -> monitor
- browse -> extract -> summarize

## Selection rules

- Keep the chain as short as possible.
- Label each step by role, not just skill name.
- Do not include a skill unless it owns a real phase transition.
- If one workflow skill can cover the full route with good confidence, prefer it over a longer chain.
