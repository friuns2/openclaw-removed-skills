## 这是什么

一个 OpenClaw AgentSkill，解决 AI 助手在处理长任务时被阻塞、新消息无法及时响应的核心问题。

适用于：所有 OpenClaw 用户、所有通道（飞书/企微/Discord等）、所有任务。

---

## 核心三层机制

### 第一层：主会话永远空闲
默认全部 spawn，主会话只做即时响应。判断标准：不知道多久 → 默认 spawn。

**主会话唯一执行（纯读）：**
- 读文件内容（cat/head/read）
- 查配置状态（openclaw config get / status）

**任何涉及写入的操作，全部 spawn：**
- 编辑文件（write/edit）→ spawn
- git 提交推送 → spawn
- 创建目录/移动文件 → spawn
- 运行脚本/命令 → spawn
- API 调用 / 文件搜索 / 编译构建 / 量化回测 / 论文抓取

### 第二层：新消息触发立即评估
收到新消息时，无论主会话当前在做什么，立即中断评估。新消息独立处理，不排队等待。

### 第三层：主动推送取代被动查询
子 agent 完成后立即通知用户，不等用户来问。

## 安装方式

### 方式一：直接安装 .skill 文件
```
openclaw skills install [文件路径]
```

### 方式二：从 GitHub Release 下载
进入 [Releases](https://github.com/colinxu1-sys/nonblocking-tasks/releases) 页面，下载最新的 `.skill` 文件，然后：
```
openclaw skills install /path/to/nonblocking-tasks.skill
```

### 方式三：手动安装
1. 克隆本仓库
2. 把 `nonblocking-tasks/` 目录复制到你的 skills 目录

---

## 安装后自动配置

1. `active_tasks.md` — 任务追踪文件
2. `HEARTBEAT.md` — 心跳检查规则（每 2 分钟）
3. Cron 任务 `nonblocking-task-monitor` — 每 2 分钟触发长任务检查
4. `AGENTS.md` / `MEMORY.md` — 规范章节更新

## 远程协助场景

当你的 AI 助手在处理一个长任务（如编译/部署/训练）时：
- 你可以随时发新消息，不会被阻塞
- AI 会立即响应你，同时在后台继续处理任务
- 任务完成后主动推送结果给你

---

## 适用人群

- 需要 AI 同时处理多个任务的用户
- 经常跑量化回测、代码编译、模型训练等长任务的用户
- 希望 AI 响应及时、不卡顿的用户
